# Variety Gate（同じ投稿が並ぶのを防ぐゲート）

目的: 工場ラインでも「文量・構成・フック」が被らないようにする

## バッチ分布（30本想定）
- 2〜3行（パンチ型）: 6本
- 4〜5行（意見型）: 9本
- 6〜8行（プロセス/失敗型）: 10本
- 9行以上（深掘り型）: 5本

## フックの種類（最低6種類）
以下のフックは**同一の型が3連続しない**ようにする
- 逆張り断言型（例: 〜は間違い）
- 失敗/反省型（例: 俺はここで詰んだ）
- 数字/実測型（例: 3回、10分、20語）
- 決定ログ型（例: こう決めた）
- 仕組み/設計型（例: こう組んだ）
- 宣言/覚悟型（例: 俺はこれをやる）

## テンション種別の分布（意外性/矛盾/不安）
- tension_tag は全投稿必須
- 30本想定の配分目安
  - surprise: 10本
  - contradiction: 10本
  - anxiety: 10本
- 同一タグが3連続したらNG

## 重複チェック
- 1行目の先頭12文字が同一の投稿が3本以上ある場合 → NG
- 末尾の語尾が連続で一致する場合 → NG

## レーン別の最低条件
- レーンA（ドラマ）: 悩み→判断→結果→未完了が入る
- レーンB（断定）: 断定 + 根拠1つ（未完了は不要）
- レーンC（実験）: 数字 + 固有名詞
- レーンD（共感/日常）: 温度感 + 小ネタ

## 返信/プロフィール遷移の運用条件（返信重視週）
- 返信トリガー投稿（A/D）を**日次2本以上**
  - 低コスト回答（A/B選択 or 一行回答）が入ること
- プロフィール遷移投稿（B/C）を**日次1本以上**
  - 「続き/次の一手/固定ポスト」を暗示すること

## CTAルール
- CTAは**自然に入る時だけ**
- 無理なCTAは内容破綻とみなして修正

## 運用
- Phase2.5後に Variety Gate を通過したものだけを review へ移動
- NGはドラフトに戻し、構成と文量を組み替える
