## 現状の課題（v1.0時点）

従来の戦略は**1話完結型（ちびまる子ちゃん型）**になっており、以下の問題がある：
- フォローしなくても1投稿で完結 → フォロー動機が弱い
- 「次どうなる？」がない → 連続性がない
- 情報提供型 → エンタメ要素がない
- **投稿数が少ない**（3.4件/日 vs 目標5件）
- **リプライ数が極端に少ない**（1.1件/日 vs 目標30件）

## 戦略の再設計

### 1. プロセスエコノミーの導入

**コンセプト**: 「複数事業を1人で回すOS」を作る**過程**をコンテンツ化

| 要素 | 従来 | 新戦略 |
|------|------|--------|
| 形式 | 1話完結（Tips/ノウハウ） | 連続ドラマ（過程の公開） |
| 視聴者の立場 | 観客 | 当事者・参加者 |
| フォロー動機 | 「役立つ情報があるかも」 | 「この挑戦がどうなるか見届けたい」 |

**ストーリーライン案**:
- 「AIエージェントで事業の80%を自動化する」という挑戦
- 毎週の進捗・失敗・学びをリアルタイム公開
- 数値（自動化率、稼働時間削減等）を定期報告

---

### 2. ワンチャン感の方程式

```
目標 × 正しい弱者 × 特殊スキル = バズ
```

**現在の設計**:
- **目標**: 複数事業をAIエージェントで自動運転化（80%自動化）
- **弱者**: 1人社長、専任チームなし、資金調達なし
- **特殊スキル**: AI実装力（Claude Code/MCP/Agentの実戦経験）

**ワンチャン感の調整**:
- 「絶対できそう」でも「絶対無理」でもない中間地点
- 「AIで自動化」は可能性を感じさせるが、「80%」は挑戦的

---

### 3. 逆ペルソナ設計（ど素人ホテル式）

**従来のペルソナ**: 経営者・事業責任者・AIエンジニア
→ 問題：この層に刺さる情報は、多くの人にとって「関係ない」

**逆ペルソナ**: 「事業OS」「AI実装」に全く興味がない人
→ この人が離脱しない構造 = 誰でも見る = AIが良質と判断

**具体的な変換**:
| 情報型（伸びない） | エンタメ型（伸びる） |
|-------------------|---------------------|
| 「RACIを導入しました」 | 「社員がまた辞めました」 |
| 「会議を自動化しました」 | 「会議に出るのを禁止されました」 |
| 「タスク管理を一本化」 | 「タスクが3つのツールに散らばって詰んだ話」 |

---

### 4. 台本構造：誤解→共感→回収

**3ステップ構造**:
1. **誤解を生む**（冒頭で止める）
   - 例：「もう辞めます」「破綻しました」「禁止されました」
2. **共感でつなぐ**（視聴者の気持ちを代弁）
   - 例：「ここまでやってきたけど、今回は無理かも」
3. **回収で納得させる**（真実を明かす）
   - 例：「辞めるのは◯◯の話でした」

**投稿テンプレート**:
```
【フック】常識破壊・誤解を生む一言
（続きを読む）
【共感】悩み・葛藤の過程
【回収】実は◯◯でした + 学び
【CTA】問いかけで締める
```

---

### 5. キャッチフレーズ変換術

日常を「事件」に変換する

| 本来の内容 | 変換後（事件化） |
|-----------|-----------------|
| 新しいツールを導入した | 「このツールを使わなければ、今頃倒産していた」 |
| 業務フローを整理した | 「社員に『それ、やらなくていいです』と言った結果」 |
| AIで自動化を進めた | 「人間がやる仕事がなくなりました」 |
| 会議を減らした | 「会議を『違法』にしました」 |
| タスク管理を統一した | 「全員のTodoを強制的に1つに集約したら起きたこと」 |

---

### 6. 参加型の仕組み

視聴者を「観客」から「当事者」に変える

**具体施策**:
- [ ] OS設計のアンケート（「どっちの構造がいい？」）
- [ ] 失敗事例の投票（「次に試すべきはどれ？」）
- [ ] テンプレート名のネーミング募集
- [ ] 「これ自動化できる？」お題募集

**効果**:
- 「自分もこの挑戦の一部」という感覚
- コメント・シェアが伸びる
- 関係人口が増加 → 商談・採用に転換

## ガードレール（更新）

### 禁止
- ツール感想だけ（必ず「OSのどこが変わるか」を1行）
- 一般論だけ（必ず自分の体験・数値を添える）
- **1話完結で終わらせる**（次回への伏線を入れる）

### 必須
- **誤解を生むフック**（冒頭で止める）
- **回収で納得させる**（釣りで終わらせない）
- 問いかけCTAで締める

## 発信前チェックリスト（必須）

### 軸足チェック（最優先）

**エッジの効いた発言をする前に、必ず確認する**

- [ ] このトピックは自分の専門領域か？（x_account_profile.md の「専門領域」に該当するか）
- [ ] この批判・断定は自分の経験・実績に基づくか？
- [ ] 反論された場合、根拠を示せるか？
- [ ] 軸足から外れた発言をしていないか？

**判定基準**:
- ✅ 軸足の範囲内 → エッジを効かせてOK（覚悟度を上げる）
- ⚠️ 軸足のグレーゾーン → 「〜な経験がある」程度に留める
- ❌ 軸足の外 → エッジを効かせない（一般論、引用、質問形式に）

**NG例**:
```
AI研究者たちは、実装を知らなすぎる（❌ 軸足の外: AI研究の専門外）
```

**OK例**:
```
俺はAIツールを実装してきたが、完全自動化は危険だと思ってる（✅ 軸足の内: 実装経験あり）
```

## 参考：DIL川岸式「集客インフラ化」戦略

出典: [川岸宏司 noteより](https://note.com/onebookof_mag/n/n2cca1be3fd55)

### 核心の転換：Xは集客装置ではなく「接点設計」

従来のチラシ→HP→フォーム導線ではなく、**認知→理解→信頼→行動**の継続的サイクルを構築する。

### Xが最適な3つの理由

| 理由 | 解説 |
|------|------|
| **①接触頻度の構築** | 「見慣れ」と「第一想起」を毎日のテキスト投稿で実現。"困った→あの人"という反射を目指す |
| **②タイムラインが取扱説明書化** | NG事項や価値観を明文化→合わない顧客が事前脱落。「会う前に人となりを判断できる」 |
| **③BtoB意思決定への適性** | 「この人に任せて大丈夫か」という不安解消がカギ。リプライ対話で信頼醸成 |

### アカウント設計5層構造

```
①アイコン：認知（モノクロで統一など、記憶に残る）
②プロフィール：事業概要・権威性
③固定ポスト：事業内容の詳細
④日常投稿：事例・FAQ・レビュー・想い
⑤企画投稿：限定オファー
```

### 顧客心理フロー別施策

| 段階 | 施策 | 目的 |
|------|------|------|
| **認知** | チラシQRをX代表アカウントへ誘導 | HPより軽いCV |
| **理解** | 毎日投稿で継続接触 | 検討中顧客の不安を「毎日1ミリ溶かす」 |
| **信頼** | 赤裸々な失敗や苦悩の共有 | 憧れより共通項が有効 |
| **行動** | 多角的な投稿テーマへの接触 | 「関心のバケツから水が溢れる」状態 |

### 重視すべき指標

**フォロワー数より「有効商談件数」**

- 月10〜20件の無料相談で実質的なKPI整合性
- 破談要因が「価値観ミスマッチ」ではなく「KPI不合致」のみという品質を目指す

### 佐藤への適用

| DIL川岸式の原則 | 佐藤への適用 |
|----------------|-------------|
| 見慣れ→第一想起 | 毎日投稿で「事業OS→さとけい」の反射を構築 |
| タイムライン＝取説 | NG事項・価値観を投稿で明文化、合わない顧客の事前フィルタ |
| 不安を1ミリ溶かす | 失敗談・苦悩を赤裸々に公開し共感を獲得 |
| 5層構造 | アイコン統一、プロフィール権威性、固定ポストで事業説明 |
| KPI＝有効商談数 | フォロワー数より「DM→商談転換率」を追う |

## 投稿テンプレート（v2.0）

### 実験ログ型（数字必須）

```
[やったこと]を[期間]やってみた

結果：
・[数字1]
・[数字2]
・[数字3]

[一番の学び/気づき]

みんなは[関連する問いかけ]？
```

**例**:
```
コミットメッセージの自動生成を1週間やってみた

結果：
・作業時間 3分→30秒に短縮
・コミット数 1.5倍に増加
・「feat/fix」の判断ミス 0件

一番の学び：
「何を変えたか」じゃなく「なぜ変えたか」をAIに聞くのが正解だった

みんなはコミットメッセージ、どうしてる？
```

### エンタメ型（誤解→共感→回収）

```
[誤解を生むフック（常識破壊）]

[共感パート：悩み・葛藤]

[回収：実は◯◯でした + 学び]

[問いかけCTA]
```

**例**:
```
コミットメッセージ、書くのやめた

「feat: 」「fix: 」って毎回考えるの、マジで時間の無駄だった

今はAIに「このdiff、どういう変更？」って聞いて、
返ってきた文章をそのままコミット

結果：コミット作業 3分→30秒

みんなはコミットメッセージ、どうしてる？
```
