自分の場合は期限は粗くても先に出すほうが判断が速い

多くは完成見通しが固まってから共有するよね

でも俺は逆で、曖昧なまま期限つけて出す

自分の場合は仮の期限を置くと判断基準が固定されて迷いが減るから決めるのが早くなる

自分の場合は企画Aで完成見通しがまだ半分くらいでも仮の期限を置いたら判断が前に進んだ、体感

ひとりで4事業回しつつ100日チャレンジ中、体感として期限を置くと判断が前倒しになる

一度やめたけど戻した、完成見通しが固まってからで十分と信じてた→裏切られた、判断が遅れて運用が崩れたから

固定ポスト見てほしい、開発完了が曖昧な課題の運用ルールをテンプレで置いてある
