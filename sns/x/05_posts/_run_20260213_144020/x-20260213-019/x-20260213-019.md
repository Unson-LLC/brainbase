部分診断を早く回すのが常識だと信じてた

でも裏切られた、納品相当まで詰めてから最終診断に戻したら手戻りが減った

途中診断を捨てると手戻りが減るのは、判断条件が揃う前の判定がノイズになって修正が増えるからだ

俺の定義で書くと、部分診断は途中の出来だけを見ること、最終診断は完成形の合否を見ること、納品相当は誰に渡しても使える状態のこと

俺は4事業と100日チャレンジを回しながらやっていて、判断条件を先に固定してテンプレ運用にした

具体例は、判断条件をゴール文と成功基準が同じ画面に揃っていることに固定したらブレが止まった
