### 2. 背景・文脈
**書くべき内容**:
- brainbaseでの課題：Skills（職能定義）は揃ったが、複雑なタスクで「連携不足」が発生
- 具体例：note記事執筆（Phase 1: 構成設計 → Phase 2: 本文執筆 → Phase 3: レビュー）を3人のSubagentで実行
- 問題点：Phase 1の結果をPhase 2に渡すが、Phase 3で「Phase 1の意図」が伝わらず手戻り
- 根本原因：Subagentは順次実行で、teammate同士が直接会話できない
- 解決策の必要性：「組織として協働する」仕組みが必要

**想定文字数**: 350文字

---

### 3. 実績・具体例
**書くべき内容**:
- Agent Teamsの実例（公式動画から）：
  - Example 1: 複数のサービスを調べて比較表を作成（Research teammate × 複数）
  - Example 2: UIデザイン + コード実装を並行作業（Design teammate + Code teammate）
- brainbaseでの実装案：
  - note-smart Orchestratorが3人のteammateを起動
  - Phase 1 teammate（構成設計）、Phase 2 teammate（本文執筆）、Phase 3 teammate（レビュー）
  - Phase 1とPhase 3が直接会話し、「意図の伝達」がスムーズに
- 効果：手戻りゼロ、品質向上、時間短縮

**想定文字数**: 400文字

### 4. 全体像・計画
**書くべき内容**:
- brainbaseにおける組織設計の全体像：
  - Skills = Job Description（職能定義）
  - Agent Teams = 組織のインスタンス化
  - teammate = 社員（役割を持った実行者）
  - RACI = 静的な権限定義 → Agent Teamsで「動く組織」に
- 組織化の4レベル（仕組み化の進化）：
  - Lv.1: ドキュメント化（マニュアル）
  - Lv.2: 自動化（ツール・スクリプト）
  - Lv.3: 仕組み化（プロセス・ワークフロー）
  - Lv.4: 組織化（Agent Teamsで属人性完全解消）
- Agent Teamsで実現する世界：「社員が辞めても会社が回る」

**想定文字数**: 450文字

---

### 5. 技術・手法
**書くべき内容**:
- Agent Teamsの技術的特徴：
  - 並列実行：複数のClaude Codeセッションが同時に動く
  - 独立コンテキスト：各teammateは独自のツール・ファイルにアクセス可能
  - teammate同士の直接通信：Orchestratorを介さず、相互に質問・フィードバック
- 実装手順（簡易版）：
  1. Orchestratorがタスクを分解
  2. 各teammateにSkillを割り当て
  3. teammateが並列実行し、必要に応じて直接会話
  4. Orchestratorが最終成果物を統合
- brainbaseでの活用例：
  - note-smart: 構成設計 × 本文執筆 × レビュー
  - marketing-strategy-planner: 調査 × 戦略立案 × 実行計画
  - 90day-checklist: RACI設計 × タスク分解 × マイルストーン設定

**想定文字数**: 450文字

**書くべき内容**:

- Agent Teamsの革命性：

  - RACIが「紙の上の権限図」から「実際に協働するチーム」に
- 仕組み化の最終形態（Lv.4）：
  - マニュアル不要（Skillsが実行者）
  - 引き継ぎ不要（teammateが即座に立ち上がる）
  - 社長不在で回る会社（マイケル・ガーバーの「仕組み経営」の完全実現）
- 未来の組織：「会社 = AIチーム」の時代
