## 本文

### 冒頭（これは何か）

複数のプロジェクトを同時に抱えていると、毎朝の「状況把握」だけで30分が溶けていく。

Slack、GitHub、メール、タスク管理ツール。複数のチャネルを行き来して、今日何をやるべきかを決める。この「仕事のための仕事」が、地味に生産性を削っている。

この問題を解決するために作った「brainbase」を、OSSとして公開しました。

---

### なぜ作ったのか

PMとして複数の事業・プロジェクトを同時に見ていると、こんな問題に直面します。

**1. 切り替えコストが高い**

プロジェクトAの作業をしていて、プロジェクトBの確認が必要になる。Claude Codeのセッションを切り替えて、コンテキストを再読み込みして、ようやく作業開始。この切り替えだけで5分かかる。

**2. 全体像が見えない**

Slackの未読、GitHubのPR、メールの返信待ち、タスク管理ツールの期限切れ。それぞれのツールを開かないと、全体の状況がわからない。

**3. 外出先で確認できない**

電車の中で「あのタスク、今どうなってたっけ？」と思っても、PCを開かないと確認できない。

---

### brainbaseとは

**ローカル完結のAI-firstプロジェクト管理コンソール**です。

複数プロジェクトのタスク・スケジュール・Inbox・Slackメッセージを一画面で確認できます。デスクトップでもスマホでも、どこからでもアクセス可能。

主な特徴：

- **ローカル完結**: データはすべて自分のマシンに保存。クラウドにデータを預けない
- **AI統合**: Claude Code、Cursor等のAIコーディングアシスタントと連携
- **セッション分離**: プロジェクトごとにClaude Codeセッションを分離（git worktree活用）
- **MCP統合**: Gmail、Slack、Airtable、GitHub等のMCPサーバーと連携
- **モバイル対応**: スマホ・タブレットからもアクセス可能

---

### 技術スタック

興味がある人のために、使っている技術を共有します。

- **Node.js** + **Express**: サーバーサイド
- **Vanilla JS**: フロントエンド（フレームワークなし、軽量）
- **tmux** + **ttyd**: Webターミナル
- **git worktree**: プロジェクトごとのセッション分離
- **MCP (Model Context Protocol)**: 外部サービス連携

特別な技術は使っていません。シンプルな構成で、誰でも理解・カスタマイズできることを重視しています。

### Quick Start（3分で始める）

前提条件:
- Node.js v20以上
- tmux
- ttyd
- Git

```bash
# 1. クローン
git clone https://github.com/Unson-LLC/brainbase.git
cd brainbase

# 2. 依存関係インストール
npm install

# 3. 設定ファイルコピー
cp config.example.yml config.yml
# config.yml を編集してプロジェクトを追加

# 4. 起動
npm start
```

ブラウザで `http://localhost:31013` にアクセスすれば、すぐに使い始められます。

### 自分で使った結果

brainbaseを3ヶ月使った結果、以下の変化がありました。

**状況把握**: 30分/日 → 5分/日
- 複数ツールを巡回する必要がなくなった
- 一画面で全プロジェクトの状況が見える

**プロジェクト切り替え**: 5分 → 10秒
- git worktreeでセッションが分離されている
- 切り替えボタン1つで、コンテキストごと切り替わる

**外出先での確認**: 不可 → 可能
- スマホからタスク確認・更新ができる
- 電車の中でも仕事の全体像が把握できる
