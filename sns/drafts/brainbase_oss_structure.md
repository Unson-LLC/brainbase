## ターゲットペルソナ（N=1）

**役職・立場**: 3-5事業を同時に回している事業責任者 / 1人経営者 / スタートアップCTO

**現在の課題**:
- 複数プロジェクトの状況把握だけで毎朝30分溶ける
- Slack、GitHub、メール、タスク管理ツールの切り替えコストで疲弊
- 外出先で「あのタスク今どうなってたっけ？」が確認できない
- AIを業務に活かしたいが、何から始めればいいかわからない

**noteを読む動機**:
- 同じ課題を解決した人の「具体的な方法」を知りたい
- ツールの選定に悩んでいる（何を使えばいいか）
- 自分でも実践できる手順を探している

**期待する成果**:
- 「これなら自分でも試せそう」という具体的なスタート方法
- 複数プロジェクト管理の効率化イメージが湧く
- OSSなので無料で試せる安心感

### ペルソナ像

田中さん（35歳・男性）。スタートアップで事業責任者として3つのプロダクトを同時に見ている。毎朝Slack、GitHub、Jiraを巡回して状況把握するのに30分かかり、その時点で疲弊している。Claude Codeは使っているが、プロジェクト切り替えのたびにセッションを再起動するのがストレス。「AIで効率化できるはず」と思っているが、何から始めればいいかわからない。

## アウトライン

### 1. 冒頭（約200文字）

- OSS「brainbase」を公開した宣言
- 一言で言うと「複数プロジェクトの管理カオスを解決するローカル完結のプロジェクト管理コンソール」
- なぜ作ったか→「毎朝30分溶けるのが嫌だった」（課題の端的な提示）
- GitHub URL提示

### 2. 背景・文脈（約500文字）

**「なぜ作ったのか」を3つの課題で説明**:

1. **切り替えコストが高い**
   - プロジェクトAからBへの切り替えで5分かかる
   - Claude Codeのセッション再起動がストレス
   - git worktreeで解決した話への伏線

2. **全体像が見えない**
   - Slack、GitHub、メール、タスク管理ツールが分散
   - 「今日何をやるべきか」を決めるのに30分
   - 一画面で全部見えれば解決

3. **外出先で確認できない**
   - 電車の中で「あのタスク」を確認したい
   - PCがないと何もできない状態
   - モバイル対応で解決

### 3. brainbaseとは（約400文字）

**主要機能の紹介**:

- **ローカル完結**: データは自分のマシンに保存、クラウド不要
- **AI統合**: Claude Code、Cursor等との連携
- **セッション分離**: git worktreeでプロジェクトごとにセッション分離
- **一元管理**: タスク、スケジュール、Inbox、Slackを一画面で
- **モバイル対応**: スマホからもアクセス可能
- **MCP統合**: Gmail、Slack、Airtable等と連携

### 4. 使った結果（約400文字）

**Before/Afterの具体的な数字**:

- **状況把握**: 30分/日 → 5分/日（83%削減）
- **プロジェクト切り替え**: 5分 → 10秒（97%削減）
- **外出先確認**: 不可 → 可能

**失敗談（プロセスエコノミー）**:
- 最初はフレームワーク（React等）を使おうとして複雑になった
- Vanilla JSに切り替えてシンプルになった
- 「シンプルに保つ」という学び

### 5. 技術スタック（約300文字）

**興味がある人向けの技術情報**:

- Node.js + Express（サーバー）
- Vanilla JS（フロントエンド、フレームワークなし）
- tmux + ttyd（Webターミナル）
- git worktree（セッション分離）
- MCP（外部サービス連携）

「特別な技術は使っていない。シンプルな構成で誰でもカスタマイズ可能」を強調

### 6. 将来の展望（約400文字）

**ビジョンを語る**:

- **短期**: AI PM機能（Slackでタスク依頼→自動登録→リマインド）
- **中期**: CEO Agent（全プロジェクトのAI PMを統括、30分対話で全事業把握）
- **長期**: マネージドサービス（セットアップ不要のSaaS版）

「マネジメントを人間がやる時代、終わらせたい」という軸足との接続

### 7. まとめ（約200文字）

**箇条書きで要点整理**:

- 課題: 複数プロジェクト管理のカオス
- 解決策: brainbase（ローカル完結・AI統合）
- 結果: 状況把握30分→5分
- CTA: **GitHub → https://github.com/Unson-LLC/brainbase**

「スターをいただけると励みになります」

### 8. 次回予告（約100文字）

- 次回は「AI PM機能」の詳細
- Slackで「@mana このタスクお願い」→自動タスク化→リマインドの仕組み
