## 推奨構文

**構文**: 進捗報告型 + Why強調

### 選定理由

1. **100日チャレンジはシリーズ投稿であり、基本は進捗報告型**
   - Day X、今日やったこと、Before/After、累計進捗、次の予告
   - シンプル・具体的・再現性

2. **ただし、Day 11はWhyを強調するタイミング**
   - 初期フォロワーに「なぜやっているか」を改めて伝える
   - 新規認知層に「この人は何のためにやっているか」を理解させる

3. **誤解→共感→回収は使わない**
   - 100日チャレンジは「ストーリー不要」の進捗報告が基本
   - 毎日投稿で「誤解→共感→回収」を使うと疲弊する
   - ただし、冒頭のフックで「Why」を示すことで差別化

4. **さとり構文・階段構文は不向き**
   - 進捗報告には合わない

### 構文テンプレート

```
【100日チャレンジ Day 11】

娘との夕食の時間を死守したい

だから「デッドライン通知」を自動化した

Before: カレンダーを毎日3回チェック（15分/日）
After: Slackに自動通知（3日前・1日前・当日）

仕組み:
Google Calendar → brainbase deadline-notifier → Slack

累計: XX個/100個完了
残り: XX日

次は「○○○」に挑戦します
```
