# MaC / Automation ガイド（AIファースト実装の入口）

目的: 実行・調整をAI側に寄せ、中間管理の手作業を排除する。AGENTS.md / Cursor Rules / LLM CoS（AI Chief of Staff）など機械可読なルールをここに集約する。

## 基本方針
- 人間は戦略・創造・例外決裁に集中。実行と調整はAI/自動化へ委譲。
- ルールは機械可読で一元管理（重複・矛盾を避ける）。
- 強制すべきもの（セキュリティ/コンプラ/ブランド基準）と推奨ガイドを分離。

## 配置ポインタ
- プロジェクト固有の実行ルール: 各リポの `AGENTS.md`（Cursor Rulesも併用可）。
- チーム/全体の強制ルール: Cursor Team Rules（Enforce）で管理。
- LLM CoS（タスク分解/割当/進捗要約）: プロンプトとルールをここにリンク。

## 参考実装のメモ
- マネジメント・アズ・コード: Team > Project > User の優先順位。強制ルールは最小限のクリティカル領域に限定。
- KPI連動: AI実行率（実行工数のうちAI比率）、エンジニア比率、管理工数削減率を `ops/kpi_definitions.md` で定義。
