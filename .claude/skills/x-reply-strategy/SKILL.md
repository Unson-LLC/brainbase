# X リプライ戦略

## Triggers

- リプ欄の上位表示を狙いたいとき
- バズ投稿に便乗して認知を取りたいとき
- `/reply` と呼ばれたとき

---

## 戦略の本質

**コメント欄の上位に表示され、元投稿の読者に刺さる**

- 元投稿を否定せず「補強」する
- エビデンスベース・論理的な指摘
- 自分のプロフィールへの誘導

**Xアルゴリズムの特性**:
> リプ欄にエビデンスベースで論理が整った指摘をすると、リプ欄の一番上に表示されやすくなる

---

## 4ステップ

### Step 1: 候補選定

**条件**:
| 項目 | 基準 |
|------|------|
| 鮮度 | 3日以内（理想は24時間以内） |
| Views | 5万+ |
| Replies | 10以下（一言感想ばかり） |
| 関連度 | 自分の専門領域 |

**狙い目**:
- 100万インプ以上なのにコメント欄が薄い
- 一言感想（「すごい」「参考になる」）ばかり
- 論理的な補足が入っていない

**実行**:
```bash
python /Users/ksato/workspace/shared/_codex/common/ops/scripts/x_research/buzz_discovery.py \
  --query "Claude Code OR ChatGPT OR AI lang:ja" \
  --min-engagement 100
```

---

### Step 2: ドラフト作成

**1行目テンプレート**（元投稿との接続）:
```
補強: 「これですね」「まさに」「補足すると」
具体化: 「自分の場合は」「実際にやってみると」
実例: 「〇〇でも同じで」
```

**構成**:
```
[1行目: 元投稿との接続]
[空行]
[自分の実体験・具体例]
[空行]
[エビデンス・数字・結論]
```

**ポイント**:
- 元投稿を否定しない
- 具体的な数字・事例を入れる
- 自分の専門性が伝わる内容

---

### Step 3: 簡易レビュー（3項目）

| チェック | 基準 |
|---------|------|
| 元投稿との接続 | 補強・具体化になっているか |
| エビデンス | 数字・実体験が含まれているか |
| 軸足一貫性 | プロフィールと矛盾しないか |

**NG例**:
- 元投稿を否定する（コメント欄で嫌われる）
- 抽象論だけ（「大事ですね」等）
- 自分語りだけで元投稿と無関係

---

### Step 4: 実行

**dry-run**:
```python
from x_client import post_tweet
post_tweet(text, reply_to='<tweet_id>', dry_run=True)
```

**本番**:
```python
post_tweet(text, reply_to='<tweet_id>')
```

## 引用リポストとの使い分け

| 項目 | リプライ | 引用リポスト |
|------|---------|-------------|
| 表示場所 | 元投稿のコメント欄 | 自分のTL |
| 目的 | コメント欄上位で認知獲得 | 自分の視点を展開 |
| トーン | 元投稿を補強 | 自分の主張を展開 |
| 適したケース | 元投稿が強い | 自分の切り口がある |
