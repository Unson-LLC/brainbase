# Phase 1: Structure Design（構成設計）

## Mission

**note記事の「構成設計」を思考フレームワークで自動化**

従来のnote執筆では「何を書けばいいかわからない」「構成を考えるのに時間がかかる」という課題がありました。

このSubagentは、customer-centric-marketing-n1の思考フレームワークを使い、**「誰に」「何を」「どの構成で」**伝えるかを自動設計します。

## Context

**トピック**: [記事にしたい内容]

**参照ファイル**:
- `_codex/sns/note_strategy.md` - note戦略（AIO、ブランディング）
- `_codex/sns/x_account_profile.md` - アカウント情報・軸足
- `_codex/sns/sns_strategy_os.md` - プロセスエコノミー、ワンチャン感

## Task

以下のステップで記事構成を設計してください：

### 1. N=1ペルソナの設定

**customer-centric-marketing-n1の思考法を適用**:
- このnote記事に**最も響く**たった一人の読者を想定
- 具体的な1人を設定（統計的な平均ではない）

**具体化すべき項目**:
- 役職・立場（例: スタートアップ経営者、事業責任者）
- 現在の課題（例: PM業務に追われている、自動化したい）
- noteを読む動機（例: 深掘りしたい、実践方法を知りたい）
- 期待する成果（例: 自分でも実践できる具体的な手順）

### 2. タイトル案の作成

**タイトルの原則**:
- 30-40文字（長すぎず短すぎず）
- 逆説的・具体的（「〇〇するために△△しない」等）
- 数字を含める（100日、10個、3つのポイント等）
- 読者の課題に直結

**タイトルのパターン**:
- **逆説型**: 「マネジメントから逃げるために、100日間毎日自動化を続けることにした」
- **疑問型**: 「なぜPMの仕事の8割は人間がやらなくていいのか」
- **宣言型**: 「PM業務100個を100日で自動化する：Day 1」
- **発見型**: 「10年PMをやって気づいた、自動化すべき業務リスト」

### 3. 9セクションからの選定

note記事の9セクションから、このトピックに必要なものを選定：

| # | セクション | 目的 | 必須度 |
|---|-----------|------|--------|
| 1 | 冒頭（これは何か） | 読者に「読む価値」を伝える | ★★★ 必須 |
| 2 | 背景・文脈（なぜやるのか） | 共感できる文脈を提供 | ★★★ 必須 |
| 3 | 実績・具体例（証拠） | 信頼性を担保する数字・事例 | ★★☆ 推奨 |
| 4 | 全体像・計画（何をするのか） | 読者に「これから何が起こるか」を見せる | ★★☆ 推奨 |
| 5 | 姿勢・ルール（失敗も公開） | プロセスエコノミーの姿勢を明示 | ★★☆ 推奨 |
| 6 | 技術・手法（透明性） | 再現可能性・信頼性を担保 | ★☆☆ 任意 |
| 7 | ビジョン・哲学（独自性） | 自分だけの視点を提供 | ★☆☆ 任意 |
| 8 | まとめ（箇条書き） | 要点を整理（スキミング読者への配慮） | ★★★ 必須 |
| 9 | 次回予告（継続読者を作る） | 「次も読みたい」と思わせる | ★★☆ 推奨 |

**選定基準**:
- シリーズ記事の初回 → 1, 2, 4, 5, 8, 9 を使用
- ノウハウ記事 → 1, 2, 3, 6, 8 を使用
- 振り返り記事 → 1, 2, 3, 7, 8 を使用
- チャレンジ宣言記事 → 1, 2, 4, 5, 7, 8, 9 を使用（全部使う）

### 4. アウトラインの作成

選定したセクションごとに、書く内容の概要を記述：

```
## 冒頭
- これは「〇〇」という[チャレンジ/実験]の記録
- なぜ始めたか→「△△」というきっかけ

## 背景・文脈
- [経験/立場]として[年数]働いてきた
- 気づいたこと: 「□□」

## 実績・具体例
- すでに[N]個を実施
- 例: [具体例1], [具体例2], [具体例3]
- 定量的な成果: [時間削減、効率化等]

...
```

### 5. 期待文字数の算出

- 各セクションの概算文字数を計算
- 合計が2,500-3,000文字になるよう調整
- 足りない場合は具体例を追加、多すぎる場合はセクションを削減

## Important Notes

1. **N=1の重要性**: 統計的な平均値ではなく、**たった一人の深層心理**を掘り下げる
2. **タイトルの具体性**: 曖昧なタイトルはクリックされない。数字・逆説・具体名詞を入れる
3. **セクション選定の柔軟性**: 必ずしも全9セクションを使う必要はない。トピックに応じて選定
4. **文字数の調整**: 2,500-3,000文字が目標。短すぎるとSEO/AIOに弱く、長すぎると読まれない

## Success Criteria

- ✅ N=1ペルソナが具体的に設定されている（役職・課題・動機が明確）
- ✅ タイトルが30-40文字で逆説的・具体的
- ✅ 使用セクションが選定されている（理由付き）
- ✅ アウトラインが作成されている（各セクションの概要）
- ✅ 期待文字数が2,500-3,000文字の範囲内

このSubagentの構成設計結果が、Phase 2の本文執筆の基盤となります。
