## Triggers

以下の状況で使用：
- 1on1ミーティングを実施するとき
- 部下との定期的な対話を設計するとき
- 1on1の準備をするとき（部下・上司両方）
- 1on1の質を改善したいとき
- コーチングスキルを向上させたいとき

---

## 1on1の本質

**原則**: 部下が主役

**目的**:
- 部下の問題・関心事を共有する
- 障害を取り除く
- キャリアの相談
- フィードバック
- 信頼関係の構築

**重要な点**:
- 1on1は**部下のための時間**である
- 上司の報告会ではない
- 緊急でない問題を扱う場（保留ファイル）

---

## 頻度と時間

**頻度**:
- 最低月1回
- 理想は週1回
- タスク習熟度（TRM）が低い部下はより頻繁に

**時間**:
- 最低1時間
- 途中で打ち切らない
- カレンダーに確実にブロック

**思考パターン**:
- この部下と月1回以上1on1をしているか？
- 1時間確保できているか？
- 途中で打ち切っていないか？

---

## 準備（部下側）

**アウトライン（議題リスト）を作る**:

部下に以下を準備させる：
1. 進捗報告（完了したタスク、進行中のタスク）
2. 問題・障害（解決が必要な課題）
3. 相談事（キャリア、スキル、チーム）
4. フィードバック要望（自分の仕事への評価）

**フォーマット例**:
```
## 進捗報告
- タスクA: 完了（結果: XXX）
- タスクB: 進行中（課題: YYY）

## 問題・障害
- 問題1: リソース不足
- 問題2: 他チームとの連携

## 相談事
- キャリア: 次のステップについて
- スキル: XXXを学びたい

## フィードバック要望
- タスクAの進め方について
```

**保留ファイル**:
- 緊急でない問題を溜めておく
- 1on1でバッチ処理
- 例: 「次の1on1で聞こう」リスト

---

## 準備（上司側）

**メモを取る準備**:
- ノート、PC、議事録ツール
- 前回の1on1メモを確認
- 継続的な問題をフォロー

**質問を用意**:
- 「最近どう？」
- 「何か困っていることは？」
- 「キャリアについて考えていることは？」

**思考パターン**:
- 前回の1on1メモを確認したか？
- 質問を用意したか？
- 聴く準備ができているか？

---

## 実行（80%ルール）

**原則**: 部下が話す時間を80%確保

**タイムライン例**:
```
0-5分: アイスブレイク、雑談
5-20分: 進捗報告（部下が話す）
20-40分: 問題・障害の相談（部下が話す、上司は質問で導く）
40-55分: キャリア・スキルの相談（部下が話す）
55-60分: まとめ、アクションアイテムの確認
```

**上司の役割**:
- 聴く（80%）
- 質問する（15%）
- 提案する（5%）

**思考パターン**:
- この1on1で部下が80%話しているか？
- 自分が話しすぎていないか？
- 部下の本音を引き出せているか？

## コーチング中心

**原則**: 回答ではなく問いで導く

**NG例**:
- 部下: 「タスクAで困っています」
- 上司: 「こうすればいいよ」❌

**OK例**:
- 部下: 「タスクAで困っています」
- 上司: 「どこで困っているの？」
- 部下: 「XXXのステップで」
- 上司: 「どうすればいいと思う？」
- 部下: 「YYYを試してみます」
- 上司: 「それいいね。他に必要なサポートは？」✅

**コーチング質問集**:
- 「どう思う？」
- 「どうすればいい？」
- 「何が障害になっている？」
- 「どんなサポートが必要？」
- 「他に選択肢は？」
- 「それを試したらどうなる？」

**思考パターン**:
- 回答ではなく質問をしているか？
- 部下自身に考えさせているか？
- コーチング質問を使っているか？

## メモを取る

**原則**: 上司がメモを取る

**理由**:
- 次回の1on1で継続フォロー
- アクションアイテムの記録
- 進捗の可視化

**メモの内容**:
- 日付
- 議題
- 部下の発言（要約）
- アクションアイテム
- 次回フォロー事項

**フォーマット例**:
```
# 1on1 with [部下名] - 2025-12-31

## 進捗報告
- タスクA完了
- タスクB進行中（課題: YYY）

## 問題・障害
- リソース不足 → アクション: XXXを調整

## 相談事
- キャリア: 次のステップについて
  - 本人の希望: ZZZ
  - アドバイス: AAA

## アクションアイテム
- [ ] 上司: XXXを調整（期限: 1週間）
- [ ] 部下: YYYを試す（期限: 次回1on1まで）

## 次回フォロー
- タスクBの進捗
- リソース調整の結果
```

**思考パターン**:
- メモを取っているか？
- アクションアイテムを明確にしたか？
- 次回フォロー事項を記録したか？

## 保留ファイルでバッチ処理

**原則**: 緊急でない問題を「保留」ファイルにバッチ処理

**保留ファイルの使い方**:
- 部下が「次の1on1で聞こう」リストを作る
- 上司も「次の1on1で話そう」リストを作る
- 1on1で一気に処理

**例**:
```
## 保留ファイル（次の1on1で）

### 部下側
- [ ] キャリアパスについて相談
- [ ] スキルアップの方向性
- [ ] チーム内の役割分担

### 上司側
- [ ] プロジェクトXのフィードバック
- [ ] 新しいツールの導入について
- [ ] チームビルディングのアイデア
```

**メリット**:
- 緊急でない問題をその場で解決しようとしない
- 1on1で集中して話せる
- 部下の時間を奪わない

**思考パターン**:
- 緊急でない問題を保留ファイルに入れているか？
- 1on1でバッチ処理しているか？
- その場で解決しようとしていないか？

## よくある失敗パターン

### 失敗1: 上司が80%話す

**症状**:
- 上司の報告会になる
- 部下が受け身
- 本音が出ない

**対処**:
- 質問で始める
- 部下に話させる
- 聴くことに集中

### 失敗2: 緊急の問題ばかり扱う

**症状**:
- 火消しばかり
- キャリア・スキルの話ができない
- 信頼関係が築けない

**対処**:
- 緊急の問題は別途対応
- 1on1は緊急でない重要な問題に使う
- 保留ファイルでバッチ処理

### 失敗3: アクションアイテムが曖昧

**症状**:
- 「考えておきます」で終わる
- 次回フォローできない
- 進捗しない

**対処**:
- アクションアイテムを明確にする
- 担当者と期限を決める
- メモに記録

### 失敗4: 時間が短い・不定期

**症状**:
- 30分で打ち切り
- 月1回もできない
- 信頼関係が築けない

**対処**:
- 最低1時間確保
- カレンダーにブロック
- 月1回以上は確実に実施

## 1on1の効果測定

**指標**:
- 部下の発言時間（80%以上か？）
- アクションアイテムの完了率
- 部下のエンゲージメント（1on1後のサーベイ）
- 問題の早期発見（1on1で問題が出てくるか？）

**改善サイクル**:
1. 1on1実施
2. メモを取る
3. 効果測定
4. 改善点を特定
5. 次回の1on1で改善
