## Triggers

以下の状況で使用：
- 新規プロジェクトで01_strategy.mdを作成するとき
- 既存の01_strategyをレビュー・改善するとき
- 戦略骨子の標準フォーマットを確認したいとき

# 01_strategy.md テンプレート

## 標準テンプレート

```markdown
## <project> 01_strategy｜戦略骨子

### プロダクト概要（1文）
<何を（What）、誰に（Who）、どうする（How）を30〜60文字で>

### ICP（1文）
<役職・業界・規模を明確に定義>

### 約束する価値（1文）
<何を一元化/自動化/最適化し（手段）+ どのような状態を実現し（状態）+ どれだけの成果を保証するか（定量）>

### 価格・前受け（内部運用方針）
<価格設定の考え方と前受けルール。brainbase原則: 前受け100%>

### TTFV目標
<Time To First Value: 顧客が最初の価値を得るまでの目標期間（具体的な日数・週数）>

### 主要KPI案
- KPI1: <指標名> - 目標値
- KPI2: <指標名> - 目標値
- KPI3: <指標名> - 目標値

### 次のアクション
- <具体的なタスク1>（担当: 〇〇、期限: 〇〇）
- <具体的なタスク2>（担当: 〇〇、期限: 〇〇）
```

## ベストプラクティス

**プロダクト概要**: 30〜60文字、何を・誰に・どうするが明確

**ICP**: 業界・企業規模・役職を含める

**約束する価値**: 定量的な成果（%、日数など）を含める

**TTFV**: 「導入完了」ではなく「価値を実感」するタイミングで定義

**主要KPI**: 3〜5個、すべて目標値付き

**次のアクション**: 「検討する」ではなく「作成する」「実施する」、担当者・期限を含める

## 保存場所

**正本**: `_codex/projects/<project>/01_strategy.md`

**プロジェクト側（参照用リンクのみ）**: `<project-repo>/docs/ops/01_strategy.md`

```markdown
# 01_strategy

正本: [_codex/projects/<project>/01_strategy.md](../../../_codex/projects/<project>/01_strategy.md)
```

## 検証チェックリスト

```
□ プロダクト概要が30〜60文字、何を・誰に・どうするが明確
□ ICPに業界・企業規模・役職が含まれている
□ 約束する価値に定量的な成果が含まれている
□ 価格・前受けルールが明記されている
□ TTFV目標が具体的な日数で定義されている
□ 主要KPIが3〜5個、すべて目標値付き
□ 次のアクションが具体的で、担当者・期限が明確
```

## 例1: SalesTailor

```markdown
## SalesTailor 01_strategy｜戦略骨子

### プロダクト概要（1文）
AI営業支援SaaSで、BtoB企業（50〜300人規模）の営業チームに、ナレッジ一元化と商談分析で成約率を2倍にする機能を提供。

### ICP（1文）
従業員50〜300人のBtoB SaaS企業で、営業組織（5〜20人）を管理し、成約率・育成速度の向上を求めるセールスマネージャー。

### 約束する価値（1文）
営業ナレッジ・商談履歴を一元化し、新人が3日で商談できる状態を実現、成約率を平均30%向上させ、マネージャーの育成工数を80%削減。

### 価格・前受け（内部運用方針）
月額5万円〜（ユーザー数課金）、前受け100%、契約時に3ヶ月分（15万円〜）を一括支払い。

### TTFV目標
導入から7日以内に最初の営業ナレッジを登録・共有し、チーム全体で活用開始。14日以内に最初の商談分析レポートを生成。

### 主要KPI案
- MRR（月次経常収益）: 500万円
- 顧客継続率: 95%以上
- NPS（顧客推奨度）: 50以上
- 商談成約率向上: 平均30%アップ
- 新人育成期間: 3日以内

### 次のアクション
- 02_offer/pricing.md を作成し、価格表とパッケージを明確化（担当: keigo、期限: SALESTAILOR-W3）
- 主要KPIのダッシュボードをLooker Studioで構築（担当: keigo、期限: SALESTAILOR-W4）
```

## 例2: brainbase

```markdown
## brainbase 01_strategy｜共通リポジトリOS

### プロダクト概要（1文）
全事業を統一OS（判断基準・テンプレ・KPI・RACI・ブランド資産）で動かし、創業者が現場にいなくても週次30分レビューだけで継続運転できる状態をつくる。

### ICP（1文）
UNSON / BAAO / SalesTailor / TechKnight / Zeims / Aitle など、複数事業を同時に扱い、引き継ぎ・再現性・立ち上げ速度が求められる GM / PO / PM / 社内オペレーション担当。

### 約束する価値（1文）
ナレッジ・判断基準・タスク・ブランド資産を brainbase に一本化し、01–05戦略・RACI・テンプレ・KPIを標準化することで、「90日で新規事業を仕組み化」「人が抜けても運営が止まらない」状態を保証する。

### 価格・前受け（内部運用方針）
前受け 100%（例外なし）。情報はすべて `_codex` に集約し、タスクは `_tasks/index.md` を唯一の正とする。

### TTFV目標
新規事業は 90日以内に 01–05（戦略）・RACI・ブランドガイド・KPI・会議リズム（週次/四半期）・前受けルール をフル実装し、自律運転できる状態をつくる。

### 主要KPI案
- タスク一本化率（_tasks への集約率）: 90%以上
- 01–05 充足率（事業ごとの OS 完成度）: 100%
- RACI / AGENTS 運用率（役割定義と行動ルールの適用度）: 80%以上
- ブランドガイド整備率（org 単位）: 100%
- 引き継ぎリードタイム（新規メンバーが理解→運用できるまでの日数）: 7日以内
```

## よくある失敗

**❌ 曖昧な表現**（例: 営業を支援するツールです）
→ ✅ 何を・誰に・どうするを明確に

**❌ 長すぎる**（100文字超）
→ ✅ 30〜60文字で要点を絞る

**❌ 定量性なし**（例: 営業がもっと楽になります）
→ ✅ %、日数など定量的な成果を含める
