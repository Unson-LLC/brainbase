# 01_strategy.md テンプレート

新規プロジェクトの戦略骨子（01_strategy.md）を作成する際の標準テンプレートとベストプラクティスです。

## Instructions

### 1. 標準テンプレートを使用

以下のテンプレートに従って作成してください：

```markdown
## <project> 01_strategy｜戦略骨子

### プロダクト概要（1文）
<1文で製品・サービスを説明>

### ICP（1文）
<理想顧客像を1文で明確に定義>

### 約束する価値（1文）
<顧客に提供する価値を1文で表現>

### 価格・前受け（内部運用方針）
<価格設定の考え方と前受けルール>

### TTFV目標
<Time To First Value: 顧客が最初の価値を得るまでの目標期間>

### 主要KPI案
- KPI1: <指標名> - 目標値
- KPI2: <指標名> - 目標値
- KPI3: <指標名> - 目標値

### 次のアクション
- <次に取り組むべきタスク1>
- <次に取り組むべきタスク2>
```

### 2. ベストプラクティスに従う

**プロダクト概要（1文）**:
- 文字数: 30〜60文字が理想
- 含めるべき要素: 何を（What）、誰に（Who）、どうする（How）
- 良い例: 「AI営業支援SaaSで、BtoB企業の営業チームに、商談成約率を2倍にする分析機能を提供」

**ICP（1文）**:
- 具体性が命: 役職・業界・規模を明確に
- 良い例: 「従業員50〜300人のBtoB SaaS企業で、営業組織を管理するセールスマネージャー」

**約束する価値（1文）**:
- 定量的な成果を含める: 「〇〇%向上」「〇日で実現」など
- フレームワーク: 何を一元化/自動化/最適化し（手段）+ どのような状態を実現し（状態）+ どれだけの成果を保証するか（定量）

**価格・前受け**:
- brainbaseの原則: 前受け100%（例外なし）
- 記載内容: 価格帯、前受けの割合、支払いタイミング

**TTFV目標**:
- 具体的な日数/週数: 「7日以内」「初回商談から30日以内」など
- 顧客視点で定義: 「導入完了」ではなく「価値を実感」するタイミング

**主要KPI案**:
- 3〜5個が理想
- 必ず目標値を含める

**次のアクション**:
- 具体的なタスク: 「検討する」ではなく「作成する」「実施する」
- 担当者を明記: RACI の Responsible を記載
- 期限を含める

### 3. 保存場所

**正本（必ず編集する場所）**:
```
_codex/projects/<project>/01_strategy.md
```

**プロジェクト側（参照用リンクのみ）**:
```
<project-repo>/docs/ops/01_strategy.md
```

内容: 以下のリンクのみを記載
```markdown
# 01_strategy

正本: [_codex/projects/<project>/01_strategy.md](../../../_codex/projects/<project>/01_strategy.md)
```

### 4. 検証チェックリスト

作成後、以下をチェック：

```
□ プロダクト概要が30〜60文字で、何を・誰に・どうするが明確
□ ICPに業界・企業規模・役職が含まれている
□ 約束する価値に定量的な成果（%、日数など）が含まれている
□ 価格・前受けルールが明記されている
□ TTFV目標が具体的な日数で定義されている
□ 主要KPIが3〜5個で、すべて目標値付き
□ 次のアクションが具体的で、担当者・期限が明確
□ 保存場所が _codex/projects/<project>/ である
□ プロジェクト側に参照リンクを配置した
```

## Examples

### 例1: SalesTailor

```markdown
## SalesTailor 01_strategy｜戦略骨子

### プロダクト概要（1文）
AI営業支援SaaSで、BtoB企業（50〜300人規模）の営業チームに、ナレッジ一元化と商談分析で成約率を2倍にする機能を提供。

### ICP（1文）
従業員50〜300人のBtoB SaaS企業で、営業組織（5〜20人）を管理し、成約率・育成速度の向上を求めるセールスマネージャー。

### 約束する価値（1文）
営業ナレッジ・商談履歴を一元化し、新人が3日で商談できる状態を実現、成約率を平均30%向上させ、マネージャーの育成工数を80%削減。

### 価格・前受け（内部運用方針）
月額5万円〜（ユーザー数課金）、前受け100%、契約時に3ヶ月分（15万円〜）を一括支払い。

### TTFV目標
導入から7日以内に最初の営業ナレッジを登録・共有し、チーム全体で活用開始。14日以内に最初の商談分析レポートを生成。

### 主要KPI案
- MRR（月次経常収益）: 500万円
- 顧客継続率: 95%以上
- NPS（顧客推奨度）: 50以上
- 商談成約率向上: 平均30%アップ
- 新人育成期間: 3日以内

### 次のアクション
- 02_offer/pricing.md を作成し、価格表とパッケージを明確化（担当: keigo、期限: SALESTAILOR-W3）
- 主要KPIのダッシュボードをLooker Studioで構築（担当: keigo、期限: SALESTAILOR-W4）
- 営業反論対応集を03_sales_ops/objections.md に整理（担当: team、期限: SALESTAILOR-W5）
```

### 例2: brainbase

```markdown
## brainbase 01_strategy｜共通リポジトリOS

### プロダクト概要（1文）
全事業を統一OS（判断基準・テンプレ・KPI・RACI・ブランド資産）で動かし、創業者が現場にいなくても週次30分レビューだけで継続運転できる状態をつくる。

### ICP（1文）
UNSON / BAAO / SalesTailor / TechKnight / Zeims / Aitle など、複数事業を同時に扱い、引き継ぎ・再現性・立ち上げ速度が求められる GM / PO / PM / 社内オペレーション担当。

### 約束する価値（1文）
ナレッジ・判断基準・タスク・ブランド資産を brainbase に一本化し、01–05戦略・RACI・テンプレ・KPIを標準化することで、「90日で新規事業を仕組み化」「人が抜けても運営が止まらない」状態を保証する。

### 価格・前受け（内部運用方針）
前受け 100%（例外なし）。情報はすべて `_codex` に集約し、タスクは `_tasks/index.md` を唯一の正とする。

### TTFV目標
新規事業は 90日以内に 01–05（戦略）・RACI・ブランドガイド・KPI・会議リズム（週次/四半期）・前受けルール をフル実装し、自律運転できる状態をつくる。

### 主要KPI案
- タスク一本化率（_tasks への集約率）: 90%以上
- 01–05 充足率（事業ごとの OS 完成度）: 100%
- RACI / AGENTS 運用率（役割定義と行動ルールの適用度）: 80%以上
- ブランドガイド整備率（org 単位）: 100%
- 引き継ぎリードタイム（新規メンバーが理解→運用できるまでの日数）: 7日以内

### 次のアクション
- 02_offer / 03_sales_ops / 04_delivery / 05_kpi を「内部OS視点」でドラフト化（担当: keigo、期限: BRAINBASE-W3）
- RACI / AGENTS の適用状況をレビュー（担当: team、期限: BRAINBASE-W4）
- KPI 目標値の決定とスコアカード登録（担当: keigo、期限: BRAINBASE-W5）
```

### よくある失敗パターン

**❌ 失敗例1: 曖昧な表現**
```markdown
### プロダクト概要（1文）
営業を支援するツールです。
```
→ 問題: 何を・誰に・どうするが不明

**❌ 失敗例2: 長すぎる**
```markdown
### プロダクト概要（1文）
当社の製品は、営業チームの皆様が日々の業務において直面する様々な課題を解決するために開発された、最先端のAI技術を活用した革新的なソリューションであり...（100文字超）
```
→ 問題: 1文が長すぎて要点が不明

**❌ 失敗例3: 定量性なし**
```markdown
### 約束する価値（1文）
営業がもっと楽になります。
```
→ 問題: 定量的な成果が不明
