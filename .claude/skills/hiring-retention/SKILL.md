## Triggers

以下の状況で使用：
- 採用面接を実施するとき
- 面接の質を改善したいとき
- 部下から退職の申し出を受けたとき
- 優秀な人材の離職を防ぎたいとき
- 採用プロセスを設計するとき

---

## 採用面接の本質

**原則**: 面接時間の80%を候補者に話させる

**目的**:
- 候補者の能力を評価
- 文化適合性を確認
- 双方のマッチングを見極める

**重要な点**:
- 面接官が80%話すのは失敗
- 候補者の本質を引き出す
- 業務に関連する場面設定で能力を問う

**思考パターン**:
- この面接で候補者が80%話しているか？
- 自分が話しすぎていないか？
- 候補者の本質を引き出せているか？

---

## 面接の4つの領域

### 1. 技術的知識・スキル

**目的**: 専門分野の理解度と実務経験を確認

**質問例**:
- 「XXXの経験はどれくらいありますか？」
- 「YYYの技術をどう使いましたか？」
- 「最近の技術トレンドで気になるものは？」

**評価項目**:
- 専門知識の深さ
- 実務経験の幅
- 技術への関心

### 2. 実績

**目的**: 過去のプロジェクトと成果を確認

**質問例**:
- 「これまでで最も誇れるプロジェクトは？」
- 「そのプロジェクトでのあなたの役割は？」
- 「どんな成果を出しましたか？」
- 「チームへの貢献は何でしたか？」

**評価項目**:
- 成果の大きさ
- チームでの役割
- リーダーシップ

### 3. 失敗と学び

**目的**: 失敗経験と成長を確認

**質問例**:
- 「これまでで最大の失敗は？」
- 「どう乗り越えましたか？」
- 「何を学びましたか？」
- 「その学びをどう活かしましたか？」

**評価項目**:
- 失敗から学ぶ姿勢
- 成長マインドセット
- レジリエンス（回復力）

**重要**:
- 失敗経験がない人は要注意
- 挑戦していない可能性
- または失敗を認めない文化

### 4. 価値観

**目的**: 仕事への姿勢とチームワークを確認

**質問例**:
- 「どんな環境で働きたいですか？」
- 「チームワークで大切にしていることは？」
- 「キャリアの目標は？」
- 「この会社で何を実現したいですか？」

**評価項目**:
- 文化適合性
- チームワーク
- 成長意欲

## 面接の流れ（60分）

### 0-5分: アイスブレイク
- 自己紹介（面接官）
- 雑談でリラックス
- 面接の流れを説明

### 5-50分: 深掘り質問（候補者が80%話す）

**時間配分**:
- 技術・スキル: 15分
- 実績: 15分
- 失敗と学び: 10分
- 価値観: 10分

**コツ**:
- オープンな質問
- 具体例を求める
- 脱線したら軌道修正

### 50-55分: 候補者からの質問
- 「何か質問はありますか？」
- 候補者の関心を確認
- 誠実に答える

### 55-60分: まとめ
- 次のステップを説明
- 連絡のタイミング
- お礼

**思考パターン**:
- 面接の流れは適切か？
- 候補者が80%話したか？
- 4つの領域を深掘りしたか？

## 面接テクニック

### テクニック1: STAR法で深掘り

**STAR法**:
- **S**ituation（状況）: どんな状況だったか？
- **T**ask（課題）: 何が課題だったか？
- **A**ction（行動）: どう行動したか？
- **R**esult（結果）: どんな結果になったか？

**質問例**:
```
候補者: 「プロジェクトXで成果を出しました」
面接官: 「どんな状況だったんですか？」（S）
候補者: 「XXXという問題がありました」
面接官: 「あなたの課題は何でしたか？」（T）
候補者: 「YYYを解決することでした」
面接官: 「どう行動しましたか？」（A）
候補者: 「ZZZという方法を試しました」
面接官: 「結果はどうでしたか？」（R）
候補者: 「AAAという成果が出ました」
```

### テクニック2: 業務に関連する場面設定

**原則**: 実際の業務に関連する場面で能力を問う

**例**:
```
質問: 「もしこういう状況になったら、どう対応しますか？」

場面設定:
- 顧客から緊急の要望
- チームメンバーの衝突
- 技術的な障害
- 締め切り直前のトラブル
```

**評価項目**:
- 問題解決能力
- 判断力
- コミュニケーション

### テクニック3: 脱線したら軌道修正

**原則**: 候補者が脱線したら優しく軌道修正

**例**:
```
候補者: 「そういえば、前の会社では...（長い話）」
面接官: 「なるほど、それは興味深いですね。では、先ほどの質問に戻りますが...」
```

### テクニック4: 沈黙を恐れない

**原則**: 候補者が考える時間を与える

**沈黙の効果**:
- 考えをまとめる時間
- 本音が出やすい
- プレッシャーテスト

## 退職引き止めの本質

**原則**: 即座に他の仕事を止めて話を聴く

**目的**:
- 真の理由を理解
- 解決策を探る
- 引き止めの可能性を探る

**重要な点**:
- 説得ではなく理由の深掘り
- 即座に対応する
- 面談で決断を変えさせようとしない

**思考パターン**:
- 退職の申し出を受けたら即座に対応しているか？
- 真の理由を深掘りできているか？
- 説得しようとしていないか？

---

## 退職引き止めの3ステップ

### ステップ1: 即座に対応

**原則**: 他の仕事を止めて即座に話を聴く

**行動**:
- 会議をキャンセル
- 時間を確保（最低1時間）
- 真剣に向き合う

**理由**:
- 部下は覚悟を決めている
- 即座に対応しないと手遅れ
- 誠意を示す

### ステップ2: 理由の深掘り

**原則**: 説得ではなく理由の深掘り

**質問例**:
- 「何があったの？」
- 「どうしてそう思ったの？」
- 「いつからそう考えてた？」
- 「他に理由はある？」

**表面的な理由 vs 真の理由**:
```
表面的な理由        真の理由
─────────────────────────────
給与が低い          評価されていないと感じる
他社のオファー      成長機会がない
家庭の事情          上司との関係が悪い
```

**深掘りのコツ**:
- オープンな質問
- 沈黙を恐れない
- 傾聴に徹する
- 判断しない

### ステップ3: 上司とともに解決策を考える

**原則**: 面談で決断を変えさせようとしない

**行動**:
- 一度持ち帰る
- 上司と協議
- 解決策を提案
- 再度面談

**解決策の例**:
- 給与改善
- 役割変更
- プロジェクト変更
- 上司変更
- 成長機会の提供

**思考パターン**:
- 真の理由を理解したか？
- 上司と協議したか？
- 解決策を提案したか？

---

## 退職面談の流れ（60分以上）

### 0-5分: 受け止める
- 「わかった、話を聞かせて」
- 動揺を見せない
- 真剣に向き合う姿勢

### 5-40分: 理由の深掘り
- オープンな質問
- 沈黙を恐れない
- 傾聴に徹する
- 判断しない

### 40-50分: 今後の選択肢を示唆
- 「解決策を考えたい」
- 「上司と協議したい」
- 「改めて話したい」

### 50-60分: まとめ
- 次回面談の予定
- 連絡のタイミング
- お礼

**注意**:
- 面談で決断を変えさせようとしない
- 説得しない
- 時間をかけすぎない（部下も疲れる）

## よくある失敗パターン

### 失敗1: 面接官が80%話す

**症状**:
- 会社の説明ばかり
- 候補者が聴くだけ
- 能力を評価できない

**対処**:
- 候補者に質問
- 80%話させる
- 深掘り質問

### 失敗2: 表面的な質問で終わる

**症状**:
- 「XXXの経験は？」「はい、あります」で終わる
- 具体例を求めない
- 深掘りしない

**対処**:
- STAR法で深掘り
- 具体例を求める
- 「どう？」「なぜ？」

### 失敗3: 退職の申し出を軽く受け止める

**症状**:
- 「考え直して」で終わる
- 理由を深掘りしない
- 即座に対応しない

**対処**:
- 即座に対応
- 理由を深掘り
- 上司と協議

### 失敗4: 面談で説得しようとする

**症状**:
- 「残ってくれ」と説得
- 部下が引けなくなる
- 解決しない

**対処**:
- 説得しない
- 理由を深掘り
- 一度持ち帰る

## brainbaseとの連携

**task-relevant-maturity との連携**:
- 候補者のTRMを面接で評価
- 入社後のマネジメントスタイルを決定

- 退職のサインを1on1で早期発見
- 定期的な対話で信頼関係構築

- 評価面談で退職意向を察知
- フィードバックで成長機会を提供

- 採用はレバレッジを高める活動
- 優秀な人材の確保
