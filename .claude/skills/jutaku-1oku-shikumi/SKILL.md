## 受託事業の本質

### 真の商品は「プロセス」と「確実性」

受託事業の商品は成果物ではなく：
- **スコープ**: どこまで責任を持つか
- **プロセス**: どう進めるか
- **確実性**: ちゃんとたどり着く安心感

クライアントは作業時間ではなく「成功確率」を買っている。

### Professional（宣誓）の姿勢

受託を資金繰りの手段と見た瞬間、事業は失敗する。
→ 「このクライアントを勝たせる」という宣誓ができない仕事は受けない。

### 一社から仕事をもらい続ける

- **バーティカル（深掘り）**: 同じ領域で追加依頼
- **ホリゾンタル（横展開）**: 別部門への紹介

強い会社は一社との関係が深まり、契約が積み上がる。「ズルズル頼まれ続ける状態」＝信頼の証。

## 法人購買の原理

### 購買理由は2つに収束

どんなニーズも最終的には：
1. **売上を上げたい**
2. **コストを下げたい**

表面のニーズ（「LPを作りたい」）と本丸の課題（「売上UP」）は別物。

### 担当者の心理

#### 1. 失敗したくない恐怖
- 成果が出ないことより「変な業者を選んで社内評価を下げること」を恐れる
- 「ホームラン」より「三振回避」が優先される
- → 実績・権威性・社内説明のしやすさが重要

#### 2. 社内政治と説明コスト
- 上司や関係部署への説明が通りやすいか
- 稟議を通せるストーリーがあるか
- → 提案は「担当者が社内で説明しやすい形」で設計する

#### 3. 信頼・安心感
- 「この人なら任せられる」という直感
- レスポンス速度、言葉遣い、資料の丁寧さ
- → 論理より先に心理的安全性を構築

## スコープ設計の5原則

### 1. 境界線を明確にする

曖昧な契約は必ず炎上する：
- 「できる限り対応します」→ NG
- 「A・B・Cは対応、D・Eは対象外」→ OK

### 2. 段階分けで提案

全部盛りではなく選択肢を提示：
- **ミニマム**: 最低限の成果（30万円）
- **スタンダード**: 推奨プラン（80万円）
- **プレミアム**: フルサポート（150万円）

### 3. 追加対応は別見積もり

「ちょっとしたお願い」を無料で受けない。
→ 小さな追加も見積もり提示し、選択させる。

### 4. 成果物の定義を具体化

「良いLPを作る」→ 曖昧
「TOPページ1枚+下層3ページ、修正2回まで」→ 明確

### 5. 前受金の徹底

着手前に30-50%を必ず受領。
→ クライアントのコミットメントを確保し、未払いリスクを排除。

## 期待値調整の技術

### 着手前の期待値すり合わせ

プロジェクト開始前に必ず実施：
1. **ゴール確認**: 「この案件で何を達成したいか？」
2. **成功基準**: 「どうなったら成功と判断するか？」
3. **懸念事項**: 「何を不安に思っているか？」

### 中間報告で軌道修正

進捗30%・70%地点で必ず報告：
- 現状共有
- 想定との差分
- 修正提案

→ 最後の納品で「イメージと違う」を防ぐ。

### 納品時のストーリー設計

成果物と一緒に「なぜこうなったか」の背景を説明：
- 課題分析
- 採用した戦略
- 期待される効果

→ ただ渡すだけでは価値が伝わらない。

## リピート・横展開の仕組み

### リピート獲得の3要素

1. **品質**: 期待を超える成果
2. **関係性**: 担当者との信頼関係
3. **タイミング**: 次の依頼タイミングで思い出される仕掛け

### 具体的施策

| 施策 | 効果 |
|------|------|
| 月次レポート配信 | 定期接触で存在感維持 |
| 業界トレンド共有 | 「頼れる専門家」ポジション確立 |
| 納品後フォロー | 「使ってみてどうですか？」で関係継続 |
| 改善提案 | 「次はこれもできます」で次案件創出 |

### 横展開（ホリゾンタル）の型

1. **納品時に他部署の課題をヒアリング**
   - 「他に困っていることはありますか？」
   - 「〇〇部門でも同様のニーズありますか？」

2. **紹介依頼の型**
   - 「似たような課題を持つ部署があれば紹介いただけますか？」
   - 「紹介料として〇〇をお渡しします」（インセンティブ設計）

3. **関連サービスの提案**
   - LP制作 → 広告運用
   - 採用動画 → 社内研修
   - システム開発 → 保守・運用

## 営業の型

### 初回商談の流れ

1. **ヒアリング（60%）**: 課題・背景・期待を深掘り
2. **提案（30%）**: スコープと価格を提示
3. **クロージング（10%）**: 次のアクションを決める

### 見積もり提示の型

```
■ご提案内容
【スコープ】
- 対応範囲: A・B・C
- 対象外: D・E（別途見積もり可）

【納期】
- 着手日: 契約後3営業日
- 納品日: 〇月〇日

【金額】
- 総額: 80万円（税別）
- 内訳: 企画30万円 / 制作40万円 / 管理10万円
- 支払条件: 着手時50% / 納品時50%

【次のステップ】
- ご発注後、キックオフMTGを設定します
```

### クロージングの型

「いかがでしょうか？」ではなく：
- 「では〇月〇日に契約書を送りますね」（前提トーク）
- 「A案とB案、どちらが良さそうですか？」（選択誘導）
- 「来週中にキックオフMTGできますか？」（次アクション確定）

## 避けるべきアンチパターン

### 1. 値引きの個別対応

「今回だけ」の値引きは信頼を損ねる。
→ 値引きは仕組み化（早期契約割引、複数契約割引）のみ。

### 2. 口約束・非文書化

「後で言った言わない」のトラブル源。
→ すべて文書化（メール・チャット・議事録）。

### 3. 無料の追加対応

「サービスでやっておきます」は自分の首を絞める。
→ 追加は必ず見積もり提示。

### 4. 前受金なしの開始

未払いリスク、クライアントの本気度不明。
→ 着手前に必ず30-50%受領。

### 5. スコープの曖昧化

「できる限り」「可能な範囲で」は炎上の元。
→ 明確な境界線を引く。
