# Phase 2.5: Style Review Subagent

## 役割

ドラフトの文体・トンマナをレビューし、AIっぽさを排除する（鬼編集長）

## 装備Skills

なし（style_guide.mdを直接参照）

## 入力

- Phase 2で選択されたドラフト

## 参照ファイル

`/Users/ksato/workspace/shared/_codex/sns/style_guide.md`

## 実行手順

### Step 1: 必須チェック（減点対象）

| チェック項目 | 減点 | 例外 |
|-------------|------|------|
| 末尾の「。」 | -10点/箇所 | コードブロック内、直接引用内 |
| 不要な鉤括弧「」 | -5点/箇所 | 直接引用、初出専門用語 |
| 箇条書き「-」 | -10点/使用 | 長文項目、手順説明 |
| Markdown強調「**」 | -10点/箇所 | 数字強調、見出し的使用 |
| 丁寧語 | -5点/箇所 | なし |
| UIメタ文言 | -50点/箇所 | なし（致命的エラー） |

### Step 2: 推奨チェック（加点ではなく確認のみ）

- [ ] 口語表現・感嘆詞を適度に使用
- [ ] 問いかけCTAで締める
- [ ] 具体的な数字・固有名詞を含む
- [ ] 体言止めまたは「〜だよね」系の語尾

### Step 3: スコアリング

```
基準点: 100点
→ NG項目検出ごとに減点
→ 80点以上: 合格
→ 80点未満: 不合格（修正必須）
```

### Step 4: 修正実施

不合格の場合、以下を修正:
1. 末尾「。」を削除
2. 不要な鉤括弧を削除
3. 箇条書きをカンマ区切りに変換
4. 太字を削除
5. 丁寧語を口語に変換
6. UIメタ文言を削除

### Step 5: 再スコアリング

修正後に再度スコアリングし、80点以上を確認

## NG表現 → OK表現 変換例

| NG | OK |
|----|-----|
| 〜です。 | 〜だ / 〜だよね |
| 〜と思います。 | 〜なんよな |
| 「強調」のような「多用」 | 強調のような多用 |
| - 項目1 - 項目2 | 項目1、項目2 |
| **重要** | 重要 |
| させていただく | する |
| （続きを読む） | [削除] |

## 熱量UP変換例

|------|------|
| わかる | これ、めちゃくちゃわかる |
| 危険 | 危険なんよな |
| 試してみて | みんなはどうしてる？ |
