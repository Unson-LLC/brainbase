## マネジャーのアウトプットの定義

**中心的な問い**: マネジャーのアウトプットとは何か？

**答え**:
```
マネジャーのアウトプット = 自身の組織のアウトプット + 影響を及ぼす隣接組織のアウトプット
```

**重要な点**:
- 個人作業のアウトプットではない
- チーム全体のアウトプットがすべて
- 隣接組織（他部門）への影響も含む

**思考パターン**:
- 自分の今日の活動は、チームのアウトプットにどう貢献したか？
- 個人作業に時間を使いすぎていないか？
- 他部門への影響を考えているか？

## マネジャーの4つの主要な仕事

### 1. 情報収集

**目的**: 現場を把握する、「知らないこと」を知る

**方法**:
- 口頭の雑談（廊下での立ち話、ランチ）
- レポート・数値データ
- 現場を歩く（Gemba Walk）
- 部下と話す（1on1、スタッフミーティング）

**情報収集の質を高める**:
- オープンな質問をする（「調子どう？」「何か困ってる？」）
- 聴くことに集中
- 複数のソースから情報を得る（三角測量）

**思考パターン**:
- 今週、現場を歩いたか？
- 部下と雑談したか？
- 複数のソースから情報を得ているか？

### 2. 情報提供

**目的**: 部下に目標・優先順位・方針を明示する

**提供すべき情報**:
- 目標（どこへ向かうか）
- 優先順位（何が重要か）
- 意思決定の根拠（なぜそう決めたか）
- 知識・ノウハウの共有

**情報提供の方法**:
- 1on1での対話
- スタッフミーティング
- メール・Slack
- ドキュメント化

**思考パターン**:
- 部下は目標を理解しているか？
- 優先順位を明示したか？
- 意思決定の根拠を説明したか？

### 3. 意思決定

**原則**: タイミングと内容が重要

**6つの質問**:
1. 何を決めるか？
2. いつ決めるか？
3. 誰が決めるか？
4. 誰に相談するか？
5. 誰が承認するか？
6. 誰に知らせるか？

**思考パターン**:
- この決定は今すべきか？
- 自分が決めるべきか？委任できないか？
- 誰に相談すべきか？

### 4. 役割モデル（ロールモデル）

**目的**: 行動で価値観を示す

**方法**:
- 模範行動を示す
- 部下が模倣すべき基準を体現
- 言行一致

**例**:
- 時間厳守 → 会議に遅れない
- オープンなコミュニケーション → 部下の意見を聴く
- 失敗から学ぶ → 自分の失敗を共有

**思考パターン**:
- 自分の行動は部下の手本になっているか？
- 言っていることとやっていることが一致しているか？
- 部下が模倣すべき行動を示しているか？

## レバレッジを高める活動

**ポジティブレバレッジ**:

**1. 多くの人に影響を与える言葉や行動**
- チーム全体へのメッセージ
- 仕組み化（一度作れば継続的に効果）
- ナレッジの共有（1対多）

**例**:
- 全体会議でのビジョン共有
- 業務プロセスの標準化
- ドキュメント化

**2. 適切な人事考課**
- 部下の成長を促す
- モチベーション向上
- チーム全体のパフォーマンス向上

**3. 知識の共有**
- 1対多の情報提供
- トレーニング
- ドキュメント化

**4. 仕組み化**
- 一度作れば継続的に効果
- 自動化
- テンプレート化

**ネガティブレバレッジ**:

**1. 意思決定の放棄**
- チーム全体が停滞
- 部下が動けない
- 機会損失

**2. 過度な干渉**
- 部下の自律性を奪う
- モチベーション低下
- マイクロマネジメント

**3. 不適切な評価**
- モチベーション低下
- 優秀な人材の離職
- チーム全体の士気低下

**思考パターン**:
- この活動は何人に影響を与えるか？
- 1時間の投資で何時間分の効果を生むか？
- ネガティブレバレッジを発生させていないか？

## モニタリングと権限委譲

**原則**: 委譲後も粗い段階で進捗を確認

### モニタリングの頻度

**部下のタスク習熟度（TRM）に応じて変える**:
- 低TRM: 頻繁に確認（毎日〜毎週）
- 中TRM: 定期的に確認（毎週〜隔週）
- 高TRM: 大きなマイルストーンのみ（月次）

### モニタリングの方法

**1. 定期的な1on1**
- 進捗確認
- 障害の早期発見
- サポート提供

**2. 進捗レポート**
- 週次レポート
- ダッシュボード
- 数値指標

**3. 成果物のレビュー**
- 途中段階でのレビュー
- フィードバック提供
- 方向修正

**思考パターン**:
- 委譲した仕事の進捗を把握しているか？
- 粗い段階で確認しているか？
- 部下のTRMに応じて頻度を変えているか？

## 時間管理

### リミッティングステップ

**原則**: 自分の時間の制約を見極める

**時間の使い方を分析**:
- 会議時間
- 移動時間
- 集中作業時間
- 1on1時間

**思考パターン**:
- 自分の時間のボトルネックは何か？
- 会議が多すぎないか？
- 集中作業時間を確保できているか？

### バッチ処理

**原則**: 類似タスクをまとめて処理

**例**:
- メール返信 → 1日2回まとめて
- 経費承認 → 週1回まとめて
- レビュー → 特定の時間帯にまとめて

**メリット**:
- コンテキストスイッチを減らす
- 集中力を維持
- 効率化

**思考パターン**:
- 類似タスクをバッチ処理しているか？
- コンテキストスイッチが多すぎないか？

### カレンダーにゆとりを持たせる

**原則**: 予定を詰め込みすぎない

**ゆとりの確保**:
- 緊急対応の時間
- 思考時間
- 休憩時間

**推奨比率**:
- 会議: 50%以下
- 集中作業: 30%
- ゆとり: 20%

**思考パターン**:
- カレンダーが詰まりすぎていないか？
- 緊急対応の時間を確保しているか？
- 思考時間を確保しているか？

## 週次レバレッジレビュー

**頻度**: 毎週金曜日

**ステップ1: 今週の活動を振り返る**
- 何に時間を使ったか？
- チームのアウトプットにどう貢献したか？
- レバレッジを発揮できたか？

**ステップ2: ネガティブレバレッジを特定**
- 意思決定を放棄していないか？
- 過度な干渉をしていないか？
- 不適切な評価をしていないか？

**ステップ3: 来週の計画**
- レバレッジを高める活動を優先
- ネガティブレバレッジを減らす
- 時間配分を調整

**フォーマット例**:
```
## 週次レバレッジレビュー（2025-12-31）

### 今週の活動
- 会議: 20時間
- 1on1: 5時間
- 集中作業: 10時間
- その他: 5時間

### レバレッジを発揮した活動
- ✅ チーム全体会議でビジョン共有（20人に影響）
- ✅ 業務プロセスを標準化（継続的な効果）

### ネガティブレバレッジ
- ❌ 意思決定を1週間放置（チームが停滞）
- ❌ 部下のタスクに過度に介入（自律性を奪った）

### 来週の計画
- [ ] 放置していた意思決定を完了
- [ ] 部下への介入を減らし、1on1で進捗確認
- [ ] ドキュメント化を進める（レバレッジ向上）
```

## レバレッジを高める具体的な方法

### 方法1: 1対多のコミュニケーション

**Before**:
- 部下一人ひとりに同じ情報を伝える（5人 × 30分 = 2.5時間）

**After**:
- チーム全体会議で一度に伝える（30分）
- レバレッジ: 2.5時間 → 0.5時間（5倍）

### 方法2: 仕組み化

**Before**:
- 毎回同じ質問に答える（週5回 × 10分 = 50分）

**After**:
- FAQドキュメントを作成（1時間）
- 継続的な効果（毎週50分節約）

### 方法3: 権限委譲

**Before**:
- すべての決定を自分でする（週10件 × 30分 = 5時間）

**After**:
- 部下に権限委譲（モニタリング: 週1時間）
- レバレッジ: 5時間 → 1時間（5倍）

### 方法4: トレーニング

**Before**:
- 部下が困るたびに教える（週5回 × 30分 = 2.5時間）

**After**:
- トレーニングセッション（2時間）
- 継続的な効果（毎週2.5時間節約）
