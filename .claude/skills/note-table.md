# noteテーブル挿入スキル

note.comの記事編集画面でテーブルを挿入する。Markdownテーブルは自動変換されないため、note.comの「表」機能を使う。

## 重要な制約

**note.comはMarkdownテーブル記法をサポートしていない！**

```markdown
# これはダメ！そのままプレーンテキストになる
| 列1 | 列2 |
|-----|-----|
| A   | B   |
```

## 解決策

### 方法1: note.comの「表」機能を使う（推奨）

Chrome DevTools MCPを使ってnote.comエディタの表機能を操作：

1. 挿入位置の「+」ボタンをクリック
2. メニューから「表」を選択
3. 行数・列数を指定
4. 各セルに内容を入力

### 方法2: テーブルを箇条書きに変換

テーブルの代わりに箇条書きで表現：

```markdown
# Before（Markdownテーブル）
| 分類 | 実装 |
|------|------|
| 全体設計者 | CEO Agent |
| 深掘り職人 | Research Sub-agent |

# After（note.com用）
**分類と実装**
- **全体設計者**: CEO Agent
- **深掘り職人**: Research Sub-agent
```

### 方法3: テーブルを画像として生成

複雑なテーブルは画像として生成して挿入：

```bash
# nanobanana-imageでテーブル画像を生成
/nanobanana-image "以下のテーブルを視覚的に美しい図表として生成：
全体設計者 → CEO Agent + 人間（週次30分）
深掘り職人 → Research Sub-agent
..."
```

## 記事作成時の推奨フロー

1. **テーブルを検出**: Markdownにテーブル記法があるか確認
2. **変換方法を選択**:
   - 2-3行の簡単なテーブル → 箇条書きに変換
   - 複雑なテーブル → 画像として生成
   - 比較表 → note.comの表機能を使用
3. **変換後にコピー**: テーブル記法を除去した状態で貼り付け

## Markdownテーブル検出パターン

```regex
^\|.*\|$
^\|[-:]+\|$
```

これらのパターンがあったら、上記の方法で変換が必要。

## 注意事項

- note.comエディタにMarkdownテーブルをそのまま貼り付けると、プレーンテキストとして表示される
- HTMLテーブルも同様にサポートされていない
- 表機能で作成したテーブルは編集可能だが、複雑なスタイリングはできない
