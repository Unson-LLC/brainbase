## 行動経済学・行動心理学の統合フレームワーク

### I. 影響力の7原則（チャルディーニ）

人は思考コストを節約するため、社会的ショートカット（原理）で承諾する。

#### 1. 返報性（Reciprocity）
- **本質**: 「返さねばならない」規範圧として作動
- **活用例**: 無料サンプル、お試し期間、先に価値提供
- **注意点**: 過度な押し付けは逆効果（リアクタンス）

#### 2. コミットメント/一貫性（Commitment and Consistency）
- **本質**: 小さな承諾が自己像を固定し、大きな承諾を呼ぶ
- **活用例**: フット・イン・ザ・ドア、段階的登録、宣言の公開
- **注意点**: 強制は禁物。自発的な選択が前提

#### 3. 社会的証明（Social Proof）
- **本質**: 情報不足時に「多数派を正解扱い」する安全策
- **活用例**: レビュー数、利用者数、「〇〇さんも使っています」
- **注意点**: 文化・コンテキスト依存。効果量にばらつき

#### 4. 好意（Liking）
- **本質**: 好きな人の頼みは断りにくい
- **活用例**: 共通点の強調、外見の好感度、協力関係の構築
- **注意点**: 表面的な好意は見抜かれやすい

#### 5. 権威（Authority）
- **本質**: 内容評価より「権威の記号」で服従を引き出す
- **活用例**: 専門家の推薦、資格・肩書きの提示、メディア掲載
- **注意点**: 偽の権威は信用失墜リスク

#### 6. 希少性（Scarcity）
- **本質**: 「失う恐れ」や「今だけ」で価値を増幅
- **活用例**: 限定〇〇名、期間限定、在庫残りわずか
- **注意点**: 嘘の希少性は炎上リスク（透明性必須）

#### 7. 一体感（Unity）
- **本質**: 「われわれ感」で、好意より深い帰属が承諾を押す
- **活用例**: 「同じ〇〇」の強調、コミュニティ形成、共通アイデンティティ
- **注意点**: カルト化リスク（`cult-marketing-theory`参照）

### II. ナッジ（選択アーキテクチャ）

選択設計は不可避。ならば「人の不完全さ」を前提に、自由を残したまま望ましい選択を増やす。

#### 1. デフォルト設定
- **本質**: 惰性（現状維持バイアス）を通じて強力に行動を動かす
- **活用例**: オプトイン/アウト、初期設定、推奨プラン
- **効果**: 最も強力なナッジ（臓器提供で国別40%差）
- **倫理**: 透明性・オプトアウト容易性が必須

#### 2. 選択肢の削減・整理
- **本質**: 選択肢が多すぎると意思決定疲れ（決定回避）
- **活用例**: 3択に絞る、カテゴリ分類、推奨の明示
- **注意点**: 削減しすぎると「選択肢がない」と感じる

#### 3. 情報提供の順序・タイミング
- **本質**: 情報提示＝意思決定そのものを設計している
- **活用例**: アンカリング、フレーミング、プライミング
- **注意点**: 一部のプライミング研究は再現性問題あり

#### 4. フィードバック
- **本質**: 行動の結果を即座に見せて学習を促進
- **活用例**: 進捗バー、リアルタイム集計、比較表示
- **効果**: 社会的証明と組み合わせて強化

---

### III. System 1/2（ファスト&スロー）

#### System 1（直感的・高速）
- **特徴**: 自動・努力なし・感情的・連想的
- **誤り**: ヒューリスティクス（代表性、利用可能性、アンカリング）
- **活用**: 初見のキャッチ、ビジュアル、感情訴求

#### System 2（熟慮的・低速）
- **特徴**: 努力・論理的・計算的・遅い
- **誤り**: 計画錯誤、過信、後知恵バイアス
- **活用**: 詳細説明、比較表、FAQ

#### 設計原則
1. **System 1で引っ掛ける** → **System 2で納得させる**
2. 重要な決定はSystem 2に引き渡す（プレモータム、外部類推）
3. 「直感の結論→後付けで弁護」になりやすいので注意

---

### IV. 欠乏と認知帯域（Scarcity）

#### 本質
- 欠乏（時間・金・手続き）は認知帯域を奪い、トンネリング（視野狭窄）を生む
- 「能力の低さ」ではなく「状況が作る帯域不足」

#### 設計原則
1. **手続き・オンボーディングは簡素化** → 考えなくて済む設計を優先
2. **欠乏状態では「今だけ」に最適化** → リスク管理施策（罰金・延滞）は逆効果
3. **スラック（余剰）を残す** → バッファ設計で帯域を回復

### V. 実務適用チェックリスト

#### Phase 1: ターゲット分析
- [ ] ターゲットの「悩み→判断→結果」を明確化（9セグマップ）
- [ ] 現在の心理状態（欠乏・認知負荷・情報過多）を診断
- [ ] System 1で何に反応するか特定

#### Phase 2: 介入設計
- [ ] 影響力の7原則のうち、どれが作動する場か設計
- [ ] デフォルト・選択肢削減・情報順序を最適化
- [ ] 欠乏がある場合、手続き簡素化を優先

#### Phase 3: 倫理チェック
- [ ] 透明性：目的を隠していないか
- [ ] 自律性：オプトアウト（退出）は容易か
- [ ] 操作性：「だまし」に近い設計になっていないか
- [ ] 長期影響：短期CVだけでなく、継続・副作用を考慮したか

#### Phase 4: 検証設計
- [ ] A/Bテスト前に「アンカーの汚染」を疑う
- [ ] 短期効果だけでなく、長期（継続率・満足度）も測定
- [ ] 外的妥当性：実験室の結果をそのまま輸入しない

## 注意事項（Must Read）

### 1. 再現性危機
- 一部のプライミング研究、署名位置介入等は撤回・疑義あり
- **「本の教訓＝実装仕様」ではなく「仮説カタログ＝現場検証が必要」**

### 2. 倫理的境界
- 説得と操作は紙一重
- 透明性・公知性・反証可能性なしでは正当性が崩れる
- カルト化リスク（`cult-marketing-theory`参照）

### 3. 外的妥当性
- 実験室の結果は「そのまま輸入すると誤る」
- 必ず段階的テスト（小→大）で検証

### 4. 長期効果
- 短期CVだけのKPIは破綻する
- 継続率・満足度・副作用（リアクタンス）まで追う
