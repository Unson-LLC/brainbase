# 顧客詳細メタ情報フォーマット

## 概要

顧客の詳細情報（会社概要・組織構造・キーマン・決裁ライン・接点履歴）を標準フォーマットで記述するためのスキル。

## ファイル配置

```
_codex/common/meta/customers/<customer_id>.md
```

## フロントマター（必須）

```yaml
---
customer_id: <id>           # 一意のID（英数字、ハイフン）
name: <正式名称>            # 会社の正式名称
projects: [project1, ...]   # 関連プロジェクトID
status: active|inactive     # 取引状態
updated: YYYY-MM-DD         # 最終更新日
---
```

## 本文構成

### 1. 顧客詳細 - <名称>

冒頭に重要な変更（社名変更など）があれば引用ブロックで記載。

### 2. 会社概要

テーブル形式で基本情報を記載：

| 項目 | 内容 |
| --- | --- |
| 正式名称 | |
| 本社 | |
| 事業 | |
| 規模 | |

### 3. 経営陣

関係する役員・管理職をテーブルで記載：

| 役職 | 氏名 | 担当 |
| --- | --- | --- |
| 代表取締役社長 | | |

### 4. 組織構造

ASCII図で関連部門の階層構造を表現：

```
会社名
├── 本部A
│   └── 部門A-1
└── 本部B
```

### 5. キーマン

案件に関わる重要人物をテーブルで整理：

| 氏名 | 役職 | 役割・権限 | 関係性 | 備考 |
| --- | --- | --- | --- | --- |
| | | | 主要窓口 | |

### 6. 決裁ライン

稟議フローをASCII図で表現：

```
[通常の稟議フロー]

  担当者（起案）
       ↓
  課長（承認）
       ↓
  部長（決裁）
```

金額別の決裁権限目安も記載：

```
[金額別目安]
  〜100万円  : 課長決裁
  〜1000万円 : 部長決裁
  1000万円〜 : 本部長決裁
```

### 7. 接点履歴

重要な接点を時系列で記録：

| 日付 | 内容 | 参加者 | 備考 |
| --- | --- | --- | --- |
| YYYY-MM-DD | | | |

### 8. 提案時の留意点

提案・営業時に注意すべきポイントを箇条書きで記載。

### 9. 関連リンク

外部リンク（会社HP、組織図ページなど）。

### 10. 関連ファイル

brainbase内の関連ファイルへのパス。
