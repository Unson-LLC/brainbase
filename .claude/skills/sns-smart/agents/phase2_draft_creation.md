# Phase 2: Draft Creation（投稿文作成）

## Mission

**Phase 1の分析結果を基に、ターゲットに響くドラフトを作成**

このSubagentは、sns-copy-patternsとx-infra-strategy-dilの思考フレームワークを使い、**適切な構文選択→フック→本文→CTAの構成**を実行します。

## Input from Phase 1

Phase 1のTarget Analysisから以下の情報を受け取ります：

```markdown
# Phase 1 実行結果: Target Analysis

## ターゲットペルソナ
[具体的な1人の人物像]

## 9セグマップ上の位置
[5セグ分類とブランド選好]

## 顧客心理フロー
[認知/理解/信頼/行動のフェーズ]

## 心理状態（悩み→判断→結果）
[悩み・判断・結果の推測]

## 訴求ポイント
[このトピックで何に響くか]

## 推奨構文
[階段/さとり/誤解→共感→回収]
```

**重要**: Phase 1の分析結果を必ず読み込み、ターゲットペルソナの心理状態に沿ったドラフトを作成してください。

## Context

**トピック**: [投稿したい内容]

**参照ファイル**:
- `_codex/sns/rules.md` - ガードレール、フックパターン

## Task

sns-copy-patternsとx-infra-strategy-dilの思考フレームワークを使い、以下を実行してください：

### 0. 軸足検証（エッジを効かせる前）

**批判・断定・踏み込みの前に、必ず軸足を確認する**

#### 1. 批判・断定箇所の抽出

ドラフト作成前に、以下のパターンを検出：
- 「〜は間違ってる」「〜はやめた方がいい」等

#### 2. 軸足からの発言か確認

- この批判は、自分の経験・実績に基づくか？
- 反論された場合、根拠を示せるか？
- x_account_profile.md の「専門領域」に該当するか？

#### 3. 軸足から外れている場合の修正

| 元の発言（軸足の外） | 修正後（軸足の内） |
|-------------------|------------------|
| 「AI研究者は実装を知らない」❌ | 「俺がAIツールを実装してきた経験では、完全自動化は危険だと思ってる」✅ |
| 「医療業界はDXが遅れてる」❌ | 「医療業界のDX事例を見ると、〜な課題がありそうだ」✅ |
| 「Reactは欠陥だらけ」❌ | 「俺はReactでbrainbaseを作ったが、〜な点で苦労した」✅ |

**重要**: 軸足から外れたエッジは、炎上リスク + 信頼失墜を招く。必ず軸足の範囲内でのみエッジを効かせること。

### 1. 構文パターンの選択

**sns-copy-patternsから適切な構文を選択**:

Phase 1で推奨された構文を基に、最適なパターンを選びます：

#### 階段構文（ロードマップ型）
**用途**: 学習順序、ツール導入順序、プロセス紹介
**構造**:
```
〇〇したいなら

ステップ1
↓
ステップ2
↓
ステップ3

この流れは覚えておいて
```

#### さとり構文（両面訴求型）
**用途**: ツール紹介、サービス訴求、幅広い層への訴求
**構造**:
```
〇〇な人は全員△△を使った方が良い。[ターゲット層へのメリット]。逆に□□な人も△△を使うことで[非ターゲット層へのメリット]が10倍になる。
```

#### 誤解→共感→回収（ストーリー型）
**用途**: 失敗談、学び、問題提起
**構造**:
```
【フック】誤解を生む一言（常識破壊）
（続きを読む）
【共感】悩み・葛藤の過程
【回収】実は◯◯でした + 学び
【CTA】問いかけで締める
```

### 2. フック（常識破壊）の作成

**rules.mdのフックパターンを活用**:

| パターン | 例 |
|---------|-----|
| 常識破壊系 | 「本当は内緒だけど」「暴露するけど」「実は〜」 |
| 禁止・警告系 | 「〜な人は読まないで」「〜だけは絶対やるな」 |
| 意外性系 | 「〜したら〇〇になった」「〇〇日で〜できた方法」 |
| 数字系 | 「〇〇%が知らない」「たった〇〇で」 |
| ストーリー系 | 「〜で失敗→〜で逆転」 |

**フックの鉄則**:
- 長文にする（続きを読むが出る長さ）
- 常識破壊を1行目に入れる
- 「続きを読む」を押させる疑問・緊張感を作る

### 3. 本文（悩み→判断→結果）の構成

**Phase 1の「心理状態」を活用**:

```
【悩み】
Phase 1で推測した「悩み」を具体的に描写
↓
【判断（きっかけ）】
Phase 1で推測した「判断」を示す
↓
【結果】
Phase 1で推測した「結果」を提示
```

**x-infra-strategy-dilの5層構造を意識**:
- ④日常投稿（事例・FAQ・レビュー・想い）として適切か？
- ⑤企画投稿（限定オファー）として適切か？

### 4. CTA（問いかけ）の作成

**rules.mdのガードレール**:
- ✅ 問いかけで締める（「〜な人いる？」「みんなはどうしてる？」）
- ❌ DM誘導は避ける（ハードルが高い）

**CTAの役割**:
- エンゲージメント促進（リプライ・いいね）
- 「自分も同じ」と思わせる共感喚起

### 5. ガードレールチェック

**rules.mdの必須チェック**:

- ✅ 「OSのどこが変わるか」を最低1行入れる
- ✅ 「悩み→判断→結果」型を必ず1案は含める
- ✅ 問いかけCTAで締める
- ✅ トーン: 簡潔・具体・再現性。誇張NG、数字は根拠付き
- ❌ 禁止: ツール感想だけ / 一般論だけ / 事業OSと無関係なAIニュース

**逆ペルソナチェック**:
- 「AIに興味がない人」でも離脱しない内容か？
- 情報型ではなくエンタメ型になっているか？

### 6. imp伸ばしの構造

**必ずこの構造で作成**:
```
【フック】← 常識破壊。「続きを読む」を押させる
--------（続きを読む）--------
【本文】← 悩み→判断→結果
【CTA】← 問いかけで締める
```

### 7. 2-3案の作成

**バリエーション**:
- 案1: 推奨構文で作成
- 案2: 別の構文パターンで作成
- 案3: 別のフックパターンで作成

**「悩み→判断→結果」型を必ず1案は含める**
