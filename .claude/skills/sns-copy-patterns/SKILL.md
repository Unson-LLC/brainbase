# SNS投稿 構文パターン集

SNS投稿文のコピーライティングパターン。投稿作成時に適切な構文を選択する。

## Triggers

以下の状況で使用：
- X/noteの投稿文を作成するとき
- スレッド投稿の構成を考えるとき
- フックやCTAの書き方を決めるとき

---

## 構文パターン

### 1. 階段構文（ロードマップ型）

**用途:** 学習順序、ツール導入順序、プロセス紹介

**構造:**
```
〇〇したいなら

ステップ1
↓
ステップ2
↓
ステップ3
↓
...

この流れは覚えておいて
```

**特徴:**
- 冒頭に目的（〜したいなら）
- `↓` で段階を繋ぐ
- 各段階は「〇〇を触りまくる / 書きまくる / やらせる」
- 易→難、手動→自動、学習→実践の順で積み上げ

**例:**
```
AIで面倒なことを自動化したいなら

ChatGPTを触りまくる
↓
Nano Banana Proを触りまくる
↓
n8nを触りまくる
↓
AIでGASを書きまくる
↓
CursorでTypescriptを書きまくる
↓
ChatGPT Atlasにブラウザ操作させる
↓
Manusに仕事やらせる

この流れは覚えておいて
```

### 2. さとり構文（両面訴求型）

**用途:** ツール紹介、サービス訴求、バズ狙い

**構造:**
```
〇〇な人は全員△△を使った方が良い。[ターゲット層へのメリット]。逆に□□な人も△△を使うことで[非ターゲット層へのメリット]が10倍になる。
```

**特徴:**
- ターゲット層と非ターゲット層の両方に訴求
- 「10倍」というフレーズを1回だけ使用（ベネフィット強調）
- 具体的な課題→解決を提示し「まさに私のことだ」と思わせる
- 体言止めを使用、句読点は最小限
- 漢字4文字以上の連続を避ける
- 120字以内（ハッシュタグ含め140字以内）

**フォーマット:**
```
第1文: 〇〇な人は全員△△を使った方が良い。
第2文: [ターゲット層へのメリット]。
第3文: [逆/非ターゲット層]も△△を使うことで[ベネフィット]が10倍になる。
第4文: （改行）ハッシュタグ4つ
```

**例:**
```
超深い思考ができる人は全員Geminiを使った方が良い。そういう人と相性がめちゃくちゃ良い。逆にあまり深く考えられない人はGeminiをうまく使いこなせばめちゃくちゃ深いところまで勝手に考えてくれるから10倍くらい頭良くなる。
```

```
頭の中が整理されている人は全員Eraserを使った方が良い。Eraserを使ってフローチャートを簡単に作れるようになると生産性が10倍になる。頭の中が整理されていないADHDタイプの人はEraserを使うことで頭が一気に整理されて作業漏れや見落としが減って社内での評価が10倍良くなる。
```

```
仕事ができる人は全員NotebookLMを使った方が良い。雑にファイルを突っ込むだけで資料の内容について回答してくれるRAGチャットボットが完成する。仕事ができない人もNotebook LMを使うことで作業効率が10倍になりシゴデキになる。今なら社内のヒーローになれること間違いなし。
```

### 3. 対比構文（ギャップ型）

**用途:** 常識への反論、意外性の提示

**構造:**
```
〇〇と思ってた

でも実際は△△だった

理由は〜
```

---

### 4. 告白構文（失敗共有型）

**用途:** プロセスエコノミー、共感獲得

**構造:**
```
正直に言う

俺は〇〇で失敗した
・具体例1
・具体例2
・具体例3

でも△△で変わった
```

---

### 5. リスト構文（網羅型）

**用途:** ノウハウ共有、まとめ系

**構造:**
```
〇〇する人が知っておくべき△個のこと

1. xxx
2. xxx
3. xxx
...

保存推奨
```

---

### 6. 問いかけ構文（参加型）

**用途:** エンゲージメント獲得、議論喚起

**構造:**
```
〇〇について聞きたい

A派？ B派？

俺は〇〇だと思ってる。理由は〜

みんなはどう？
```

## スレッド構成パターン

### 基本6ツイート構成

1. **フック** - 結論 or 問題提起 + 画像
2. **背景** - なぜこの話をするのか
3. **本論1** - コア概念の説明
4. **本論2** - 具体例 or 実装
5. **教訓** - 学び or 注意点
6. **CTA** - リンク or 行動喚起

---

## 使い方

投稿内容に応じて適切なパターンを選択し、構造に当てはめて作成する。

パターンは組み合わせ可能。例：
- 階段構文 + リスト構文
- 告白構文 + 対比構文
