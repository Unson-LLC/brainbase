# note記事作成統合スキル

note.comへの記事投稿を自動化する統合ワークフロー。

## 前提条件

- Claude Code
- Claude in Chrome拡張機能（ブラウザ操作用）
- GEMINI_API_KEY（画像生成用）
- macOS環境
- note.comアカウント（ログイン済み）

## 使い方

```
「〇〇についてのnote記事を作成して。挿絵を2枚生成して挿入して」
```

## 自動化フロー

### Phase 1: 記事本文の作成

1. ユーザーの指示に基づいて記事を執筆
2. **テーブル記法を検出して変換**（重要！）
3. 見出し・本文・まとめの構成

### Phase 2: テーブルの変換（自動）

**Markdownテーブルを検出したら自動変換する**

```markdown
# 変換前（Markdown）
| 分類 | 実装 |
|------|------|
| A | B |

# 変換後（note.com用）
**分類と実装**
- **A**: B
```

または、複雑なテーブルは画像として生成：
```
/nanobanana-image テーブルを視覚的な図表として生成...
```

### Phase 3: 画像生成

1. `/nanobanana-image` で概念図・挿絵を生成
2. `/tmp/` に保存
3. 生成画像をユーザーに確認

### Phase 4: note.comへ投稿

1. `https://note.com/new` を開く
2. タイトルを入力
3. 本文を貼り付け（**テーブル記法なし**）
4. `/note-image` で画像を挿入
5. タグを設定
6. 下書き保存

## テーブル変換ルール

| テーブルの複雑さ | 変換方法 |
|----------------|---------|
| 2列・3行以下 | 箇条書きに変換 |
| 比較表 | 画像として生成 |
| データテーブル | note.comの表機能を使用 |

## 例：変換パターン

### パターン1: シンプルな対応表

```markdown
# Before
| 役割 | 担当 |
|------|------|
| 全体設計者 | CEO Agent + 人間 |
| 深掘り職人 | Research Sub-agent |

# After（箇条書き）
**役割と担当**
- **全体設計者**: CEO Agent + 人間
- **深掘り職人**: Research Sub-agent
```

### パターン2: 複雑な比較表

```markdown
# Before
| 項目 | プランA | プランB | プランC |
|------|---------|---------|---------|
| 価格 | ¥1,000 | ¥2,000 | ¥3,000 |
| 機能 | 基本 | 標準 | フル |

# After（画像生成）
/nanobanana-image 以下の比較表を視覚的に美しい図表として生成：
プランA: ¥1,000・基本機能
プランB: ¥2,000・標準機能
プランC: ¥3,000・フル機能
```

## 関連Skills

- `/nanobanana-image` - 画像生成
- `/note-image` - 画像挿入
- `/note-table` - テーブル変換ガイド

## 注意事項

1. **テーブル記法は必ず変換**: そのまま貼り付けるとプレーンテキストになる
2. **画像は絶対パスで**: `/tmp/generated_image.png` など
3. **下書き確認**: 公開前に必ずプレビューで確認
