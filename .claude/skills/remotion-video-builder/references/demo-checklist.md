# Remotion Live Demo Checklist

## Goal

Convince the audience that video generation can be treated like software: editable, repeatable, and template-driven.

## Recommended 20-Minute Runbook

1. Open with the rendered output.
2. State the promise in one sentence:
   "This video is React code, so changing one prop gives us a new asset."
3. Show the project structure briefly.
4. Open the main composition.
5. Explain only these concepts:
   - composition
   - props
   - timeline
   - render output
6. Make one safe live change:
   - headline text
   - accent color
   - CTA
7. Preview the changed scene.
8. Render the updated clip.
9. End with business use cases:
   - webinar teaser
   - sales proposal opener
   - event announcement
   - short vertical social clip

## Fallback Plan

If live coding becomes unstable:

1. Switch to a prepared branch or file.
2. Show before/after code diff.
3. Render the prepared variant.
4. Continue the business explanation without debugging on stage.

## What To Avoid

- Building a complex scene from scratch on the call
- Explaining every Remotion API
- Spending time on environment setup during the live portion
- Showing long render waits without narration
