# セキュリティインシデント対応プレイブック

DialogAI CVE-2025-55182対応（2025年12月）の教訓を基に作成した、セキュリティインシデント発生時の標準対応手順。

---

## 初動チェックリスト（検知後1時間以内）

### ✅ Step 1: 状況確認（15分以内）

- [ ] インシデント検知ツールを確認（GuardDuty、CrowdStrike等）
- [ ] 影響範囲を特定（どのシステム？どのデータ？）
- [ ] 初動対応を実施（NW遮断は情セキと連携）
- [ ] 証拠保全を開始（ログ、スナップショット）

### ✅ Step 2: エスカレーション（30分以内）

**クライアント側責任者（大槻氏等）にエスカレーション:**

```
件名: [緊急]〇〇システム セキュリティインシデント発生

【状況】
- 検知日時: YYYY-MM-DD HH:MM
- 検知ツール: （GuardDuty/CrowdStrike等）
- 影響システム: 〇〇
- 初動対応: NW遮断済み/未実施

【推奨対応】
- 外部専門家（ワイドアングル等）によるフォレンジック調査
- 費用: 別途見積もり（保守契約外）

【判断期限】
- 本日中

【次のアクション】
- フォレンジック調査の実施可否判断
- コミュニケーション体制の確立
```

### ✅ Step 3: 体制確立（1時間以内）

**エスカレーションフロー:**
- **技術判断**: 自分 → チーム
- **契約判断**: 自分 → クライアント側責任者
- **クライアント対応**: 責任者に移譲

**コミュニケーション:**
- [ ] Teamsチャンネル作成・集約
- [ ] 報告頻度を合意（1日1回定時報告17時、重要事項は即時）
- [ ] 対応可能時間を明示（営業時間内2時間以内、緊急時1日以内等）

## フォレンジック調査の判断（24時間以内）

### 外部専門家が対応可能な場合

**即座に依頼を推奨:**
- 専門性、信頼性、報告書の対外的説得力が高い
- 法的証拠能力がある
- クライアントへの説明で信頼性が高い

### 外部専門家が対応不可の場合（DialogAIケース）

**選択肢を提示し、判断を仰ぐ:**

```markdown
クライアント側責任者へ

外部専門家（ワイドアングル）対応不可とのことですが、
以下の選択肢があります：

## 【A案】他社専門家に依頼（推奨★★★★★）

- **候補**: ラック、NRIセキュア、トレンドマイクロ等
- **費用**: 100-300万円
- **期間**: 1-2週間
- **結論の確度**: 95%以上（法的証拠能力あり）
- **メリット**: クライアントへの説明で信頼性が高い
- **デメリット**: 費用、期限（〇月〇日）に間に合わない可能性

## 【B案】簡易調査のみ実施（推奨★★☆☆☆）

- **費用**: XX万円（工数50h程度）
- **期間**: 3-5日
- **結論の確度**: 60-70%（限定的）
- **メリット**: 低コスト、早い、期限に間に合う
- **デメリット**:
  - 「データ漏洩なし」を断言できない
  - クライアントへの説明時、信頼性を問われるリスク
  - ログ欠損により攻撃時の通信を追跡不可
- **重要**: レポートに「簡易調査である」旨を明記

## 【C案】状況確認のみ、詳細調査は見送り

- **メリット**: 低コスト
- **デメリット**: 漏洩有無が不明確
- **推奨度**: ★☆☆☆☆

判断をお願いします。
期限が迫っている場合、B案で進めざるを得ませんが、
上記リスクをクライアントに事前説明することを推奨します。
```

### 簡易調査を実施する場合（B案）

**必ず事前に明示すること:**

1. **工数と費用の見積もり:**
```
フォレンジック調査の見積もり:
- 調査範囲: ログ分析、ファイルシステム分析、IOC抽出
- 工数: 50時間（5-6日間）
- 費用: XX万円（保守契約外、別途請求）
- 成果物: フォレンジックレポート
```

2. **リスクの明示:**
```
重要な制約事項:
- 専門家ではないため、見落としの可能性あり
- 「データ漏洩なし」を断言できない（「可能性は低い」止まり）
- クライアントへの説明時、信頼性を問われるリスク
- レポートに「簡易調査である」旨を明記
```

3. **承認の取得:**
```
上記の制約を理解した上で、実施を承認いただけますか？
承認いただけない場合、外部専門家への再依頼を推奨します。
```

## 安全宣言の基準（事前合意必須）

### 対応の3段階分類

| フェーズ | 内容 | Go基準 |
|---------|------|--------|
| **短期対応** | 緊急対応（NW遮断、脆弱性パッチ、初期ログ確認） | システム再開可能 |
| **中期対応** | 再発防止策（セキュリティ設定強化、監視体制整備） | 同じ攻撃で侵入されない |
| **長期対応** | セキュリティ強化（WAF、多層防御、ログ監視自動化） | セキュリティレベル向上 |

### 安全宣言のタイミング

**短期対応完了時点:**
```
「短期対応が完了し、システムを再開可能な状態になりました。
 ただし、中期・長期対応は継続中です。」
```

**中期対応完了時点:**
```
「再発防止策が完了しました。同じ攻撃では侵入されない状態です。
 長期的なセキュリティ強化は継続中です。」
```

## 契約条項（事前に定義すべき）

### インシデント対応の扱い

```markdown
## セキュリティインシデント対応

### 対応範囲
- 保守契約には含まれない
- 発生時は別途見積もりを提示

### 対応体制
- 初動: 状況確認、エスカレーション（2時間以内）
- 調査: 外部専門家への依頼を推奨（費用別途）
- 復旧: 工数ベースで見積もり（費用別途）

### 費用
- 緊急対応: 通常単価の1.5倍
- 休日対応: 通常単価の2倍
- 外部専門家: 実費請求

### 免責事項
- 自社による調査は「簡易調査」扱い
- 専門家による調査を推奨するが、費用判断はクライアント側
```

## 報告フロー

### 定期報告（1日1回）

**報告時刻**: 17:00（営業時間内）

**報告内容:**
- 本日の対応内容
- 明日の予定
- 課題・懸念事項
- クライアントへの確認事項

### 重要事項報告（即時）

**報告すべき事項:**
- フォレンジック調査で重大な発見があった場合
- データ漏洩の可能性が高まった場合
- 対応方針の変更が必要な場合
- 費用増加の可能性がある場合

---

## 立ち位置と権限の範囲（重要）

### 決裁可能

- 技術判断（どの調査手法を使うか、どのツールを使うか）
- brainbase運用判断
- タスク管理

### 確認必須（越権禁止）

- **契約判断**: フォレンジック調査の実施可否、費用
- **金銭判断**: 追加費用、外部専門家への依頼
- **人事判断**: 体制変更、外部リソースの起用
- **他法人CEO判断**: クライアント側の判断（安全宣言、システム再開等）

### 教訓

- 「依頼されたから受ける」ではなく、「選択肢を提示して判断を仰ぐ」
- 技術的限界やリスクを明示する
- 「無理です」「他の選択肢があります」と言える関係性の構築
