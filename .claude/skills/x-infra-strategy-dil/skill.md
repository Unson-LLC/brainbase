# DIL川岸式「X集客インフラ化」戦略

出典: [川岸宏司 note](https://note.com/onebookof_mag/n/n2cca1be3fd55)

## Triggers

以下の状況で使用：
- Xアカウントの設計・見直しをするとき
- BtoB集客のSNS戦略を考えるとき
- フォロワー数より商談・採用を重視したいとき
- アカウントの5層構造を設計するとき

---

## 核心の転換

**Xは集客装置ではなく「接点設計」**

従来のチラシ→HP→フォーム導線ではなく、**認知→理解→信頼→行動**の継続的サイクルを構築する。

---

## Xが最適な3つの理由

### ①接触頻度の構築
- 「見慣れ」と「第一想起」を毎日のテキスト投稿で実現
- 相手の脳内検索で"困った→あの人"という反射を目指す
- 例: 「モノクロのアイコンでよく思考術を発信する人」→ 第一想起獲得

### ②タイムラインが取扱説明書化
- NG事項や価値観を明文化することで、合わない顧客が事前脱落
- 「会う前に人となりを判断できる」プラットフォーム
- 破談要因が「価値観ミスマッチ」ではなく「KPI不合致」のみに

### ③BtoB意思決定への適性
- 「この人に任せて大丈夫か」という不安解消が最終決定のカギ
- リプライ機能による対話で信頼醸成
- 検討中顧客の不安を「毎日1ミリ溶かす」

## アカウント設計5層構造

```
①アイコン：認知
  └ モノクロで統一など、記憶に残る一貫性

②プロフィール：事業概要・権威性
  └ 何をしている人か + 信頼できる根拠

③固定ポスト：事業内容の詳細
  └ サービス説明、実績、お問い合わせ導線

④日常投稿：事例・FAQ・レビュー・想い
  └ 毎日投稿で接触頻度を確保

⑤企画投稿：限定オファー
  └ キャンペーン、無料相談、資料配布
```

### 各層の設計ポイント

| 層 | 役割 | チェックポイント |
|---|------|-----------------|
| ①アイコン | 認知 | 一目で識別できるか？ 統一感があるか？ |
| ②プロフィール | 権威性 | 何の専門家か明確か？ 実績が見えるか？ |
| ③固定ポスト | 詳細説明 | サービス内容が伝わるか？ CTAがあるか？ |
| ④日常投稿 | 信頼構築 | 事例/FAQ/レビュー/想いのバランスは？ |
| ⑤企画投稿 | 行動喚起 | 限定性があるか？ 参加ハードルは適切か？ |

## 顧客心理フロー別施策

### 認知段階
- **施策**: チラシQRをX代表アカウントへ誘導
- **目的**: HPより軽いCV（フォローは問い合わせより軽い）
- **ポイント**: 名刺、チラシ、メール署名にXリンク

### 理解段階
- **施策**: 毎日投稿で継続接触
- **目的**: 検討中顧客の不安を「毎日1ミリ溶かす」
- **ポイント**: FAQ、事例、よくある誤解への回答

### 信頼段階
- **施策**: 赤裸々な失敗や苦悩の共有
- **目的**: 憧れより共通項が有効（共感獲得）
- **ポイント**: 過去の失敗談、学び、価値観の明文化

### 行動段階
- **施策**: 多角的な投稿テーマへの接触
- **目的**: 「関心のバケツから水が溢れる」状態を形成
- **ポイント**: 事例×FAQ×レビュー×想いの組み合わせ

---

## 日常投稿の4テーマ

### 1. 事例
- 実際のクライアント事例（匿名可）
- Before/Afterの数値変化
- 「こういうケースでこう解決した」

### 2. FAQ
- よくある質問への回答
- 「〇〇って聞かれるけど、実は〜」
- 検討中顧客の疑問を先回りで解消

### 3. レビュー
- クライアントの声
- 第三者の評価（権威性の補強）
- 「こんなフィードバックをもらった」

### 4. 想い
- なぜこの仕事をしているか
- 価値観、NG事項の明文化
- 合わない顧客の事前フィルタリング

## 重視すべき指標

### フォロワー数より「有効商談件数」

| 指標 | 従来の考え方 | DIL川岸式 |
|------|------------|----------|
| Reach | フォロワー数 | **有効商談件数**（月10〜20件目標） |
| Quality | エンゲージメント率 | **破談率の低さ**（価値観ミスマッチ起因ゼロ） |
| Efficiency | CPA | **商談品質**（KPI不合致のみで破談） |

### 計測方法
- 問い合わせ経路を必ず確認（「Xを見て」の割合）
- 商談成立/破談の理由を記録
- 破談理由が「価値観」なら投稿内容を見直し

## フック手法

### 自慢・成果開示型
```
note記事の反応が60倍以上になっていまして…
月20件のお問い合わせ達成
```

### 問い返し型
```
インスタやTikTokじゃダメなの？
→ 経営者からの質問を想定してフック化
```

### 共感・赤裸々型
```
17歳起業時の営業地獄
チラシ配布の脂汗
→ 失敗談・苦労話を晒す
```

### 逆張り型
```
Xの強さは拡散力ではなく、関心の『インフラ化』
→ 常識への反論
```

---

## 避けるべきこと

- 自動化ツールへの過度な依存（「本当に成果出てるか？」を常に問う）
- フォロワー数を追うあまり、有効商談を見失う
- 1話完結で終わらせる（継続接触の設計がない）
- 価値観・NG事項を隠す（合わない顧客との破談が増える）

## 実績（川岸宏司の場合）

- 180社以上との協業
- 子会社3社創業
- 著書2冊出版予定
- 月20〜30件の新規契約（有効商談ベース）
