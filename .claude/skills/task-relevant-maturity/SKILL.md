## Triggers

以下の状況で使用：
- 部下への指示出しを検討するとき
- 権限委譲を判断するとき
- 部下の成長段階を評価するとき
- マネジメントスタイルを変えるべきか悩むとき
- 新しいタスクを割り当てるとき
- 1on1でTRMを評価するとき

---

## タスク関連習熟度（TRM）の本質

**定義**: 特定のタスクに対するスキル・経験・達成志向

**重要な点**:
- TRMは**特定のタスクに対するもの**であり、一般的な有能さとは別物
- 同じ人でも**タスクが変わればTRMは変わる**
- タスク特異的に評価する

**例**:
- ベテランエンジニアが新しい技術スタックを学ぶ → TRM低
- 新人が得意分野のタスクを担当 → TRM中〜高
- マネジャーが初めて採用面接をする → TRM低

**思考パターン**:
- この部下のTRMは、**このタスクに対して**どうか？
- タスクが変わったときにTRMを再評価したか？
- 一般的な有能さとタスク特異的なTRMを混同していないか？

---

## TRMの3要素

### 1. スキル

**定義**: タスクを遂行するための技術的能力

**評価項目**:
- 必要な知識があるか？
- 実務経験はあるか？
- ツールを使いこなせるか？

### 2. 経験

**定義**: 同様のタスクをこなした回数と質

**評価項目**:
- 過去に何回やったか？
- 成功体験はあるか？
- 失敗から学んだか？

### 3. 達成志向

**定義**: タスクを完遂する意欲と責任感

**評価項目**:
- モチベーションは高いか？
- 自主的に動くか？
- 責任を持って完遂するか？

**TRMの判定**:
```
TRM = スキル × 経験 × 達成志向

いずれかが低い → TRM低
すべて高い → TRM高
```

---

## TRMの3段階

### 低TRM

**特徴**:
- スキル不足
- 経験不足
- 達成志向が低い（または不明）

**例**:
- 新人
- 新しいタスクに挑戦
- 初めての技術スタック

### 中TRM

**特徴**:
- スキルはあるが経験不足
- または経験はあるがスキル不足
- 達成志向は中程度

**例**:
- 1-2回経験したタスク
- 基本スキルはあるが応用は不慣れ

### 高TRM

**特徴**:
- スキル十分
- 経験豊富
- 達成志向が高い

**例**:
- 何度もこなしたタスク
- 専門分野のタスク
- 自律的に動ける

---

## マネジメントスタイルの変化

### 低TRM → タスク志向（構造化・指示）

**原則**: 何を・いつ・どうやるかを細かく指示

**具体的な行動**:
- 詳細な指示を出す
- 頻繁なチェック（毎日〜毎週）
- 手順を明示
- 成果物のレビュー

**コミュニケーション**:
- 「こうしてください」
- 「次はこれをやってください」
- 「この手順で進めてください」

**例**:
```
タスク: 新機能Aの開発（新人エンジニア、TRM低）

指示:
1. 仕様書を読む（期限: 今日中）
2. 設計案を作成（期限: 明日）
3. 設計レビュー（明後日10時）
4. 実装開始（レビュー後）
5. 毎日夕方に進捗確認
```

**思考パターン**:
- 詳細な指示を出しているか？
- 頻繁にチェックしているか？
- 手順を明示しているか？

### 中TRM → コミュニケーション志向（支援）

**原則**: コミュニケーションと励まし、自主性を尊重しつつサポート

**具体的な行動**:
- 目標と期待値を明示
- 定期的な1on1（毎週）
- 障害を取り除く
- 励ましとフィードバック

**コミュニケーション**:
- 「どう思う？」
- 「どうすればいいと思う？」
- 「困ったことある？」
- 「いい感じだね！」

**例**:
```
タスク: 新機能Bの開発（経験2回目、TRM中）

指示:
- 目標: 新機能Bを2週間で完成
- 期待値: 品質基準Xを満たす
- 週1回の1on1で進捗確認
- 困ったらいつでも相談してね
```

**思考パターン**:
- 目標と期待値を明示したか？
- 自主性を尊重しているか？
- サポートを提供しているか？

### 高TRM → 最小限の関与（委任）

**原則**: 目標とモニタリングに専念、自律的に動ける環境を提供

**具体的な行動**:
- 目標のみ設定
- 大きなマイルストーンのみ確認（月次）
- 報告を受ける
- 障害があればサポート

**コミュニケーション**:
- 「目標はXXX、期限はYYY。あとはお任せ」
- 「何か障害ある？」
- 「順調そうだね」

**例**:
```
タスク: 新機能Cの開発（専門分野、TRM高）

指示:
- 目標: 新機能Cを1ヶ月で完成
- 月次レポートで進捗報告
- 障害があれば即座に相談
```

**思考パターン**:
- 目標のみ設定しているか？
- 自律的に動ける環境を提供しているか？
- 過度に干渉していないか？

## TRMの判定方法

### 方法1: 観察

**観察項目**:
- タスクの遂行速度
- 成果物の質
- 自主的な行動
- 質問の頻度

### 方法2: 1on1での質問

**質問例**:
- 「このタスク、何回やったことある？」
- 「どこで困りそう？」
- 「自信ある？」
- 「サポート必要？」

### 方法3: 小さなタスクで試す

**原則**: いきなり大きなタスクを任せない

**ステップ**:
1. 小さなタスクを任せる
2. TRMを観察
3. 評価
4. 大きなタスクへステップアップ

### 方法4: 自己評価

**質問**:
- 「このタスクに対する自分のTRMは？（低・中・高）」
- 「何が不安？」

**注意点**:
- 自己評価は過大評価の傾向
- 客観的な観察と組み合わせる

---

## 価値観の共有と委任

**原則**: 権限委譲には価値観とやり方の共有が不可欠

**なぜ共有が必要か**:
- 部下が独自の判断をする
- 判断基準が必要
- 価値観が合っていないと失敗

**共有すべき価値観**:
- 品質基準
- 優先順位の考え方
- 顧客への姿勢
- チームワークの重要性

**警告**: 部下に「失敗から学ばせる」ことは**顧客に授業料を払わせること**

**思考パターン**:
- 価値観を共有しているか？
- やり方を共有しているか？
- 部下が独自判断できる状態か？

---

## 部下の習熟度評価の難しさ

**マネジャーの過大評価傾向**:
- 自分のコミュニケーション能力を過大評価
- 委譲度合いを過大評価
- 部下のTRMを過大評価

**対処**:
- 客観的な自己検証
- 第三者からのフィードバック
- 成果物での評価

**思考パターン**:
- 自分の評価は客観的か？
- 過大評価していないか？
- 第三者の意見を聞いたか？

---

## タスク変更時のTRM再評価

**原則**: タスクが変わったらTRMを再評価

**例**:
```
エンジニアA（React開発: TRM高）
└─ 新タスク: Python開発 → TRM低（再評価）

マネジャーB（部門運営: TRM高）
└─ 新タスク: 採用面接 → TRM低（再評価）
```

**再評価のトリガー**:
- 新しい技術スタック
- 新しい役割
- 新しいプロジェクト
- 新しいチーム

**思考パターン**:
- タスクが変わったときにTRMを再評価したか？
- 前のタスクのTRMをそのまま適用していないか？

## TRMに応じたマネジメントマトリックス

```
TRM     スタイル             関与度    頻度           コミュニケーション
─────────────────────────────────────────────────────────────────────
低      タスク志向           高        毎日〜毎週     「こうしてください」
中      コミュニケーション   中        毎週           「どう思う？」
高      最小限の関与         低        月次           「お任せ」
```

## よくある失敗パターン

### 失敗1: 低TRMに委任

**症状**:
- 新人にいきなり大きなタスク
- 指示不足
- 失敗

**対処**:
- タスク志向で細かく指示
- 小さなタスクから始める

### 失敗2: 高TRMにマイクロマネジメント

**症状**:
- ベテランに細かく指示
- モチベーション低下
- 離職

**対処**:
- 最小限の関与
- 自律性を尊重

### 失敗3: タスク変更時にTRM再評価しない

**症状**:
- 前のタスクのTRMをそのまま適用
- 新タスクで失敗

**対処**:
- タスクが変わったら再評価
- 小さなタスクで試す

### 失敗4: 価値観を共有せずに委任

**症状**:
- 部下が独自判断で失敗
- 顧客に迷惑

**対処**:
- 価値観を共有
- やり方を共有

## TRM評価チェックリスト

### 事前評価
- [ ] このタスクに対する部下のスキルは？
- [ ] このタスクの経験回数は？
- [ ] 達成志向は高いか？
- [ ] TRMは低・中・高のどれか？

### マネジメントスタイル
- [ ] TRMに応じたスタイルを選んでいるか？
- [ ] 低TRM: タスク志向（細かく指示）
- [ ] 中TRM: コミュニケーション志向（励まし）
- [ ] 高TRM: 最小限の関与（委任）

### 委任時
- [ ] 価値観を共有したか？
- [ ] やり方を共有したか？
- [ ] 失敗から学ばせる = 顧客に授業料を払わせることを理解しているか？

### タスク変更時
- [ ] タスクが変わったらTRMを再評価したか？
- [ ] 小さなタスクで試したか？
