# WIP（作業中チェックポイント）

作業途中の状態を素早く保存します。後でsquashすることを前提とした一時コミット。

## 使い方

作業中に「一旦保存したい」と思ったら即実行。

## 実行手順

1. 現在の変更を確認
   ```bash
   git status
   git diff --stat
   ```

2. 全ファイルをステージング
   ```bash
   git add -A
   ```

3. WIPコミット
   ```bash
   git commit -m "wip: <今やっていること>"
   ```

## WIPコミットの例

```
wip: ログイン機能実装中（認証部分まで完了）
wip: APIレスポンスのパース実装中
wip: テスト追加中（半分完了）
wip: リファクタリング途中
```

## 作業完了時の整理

WIPコミットが溜まったら、最後にsquash:

```bash
# 直近3つのコミットを1つにまとめる
git rebase -i HEAD~3
```

エディタで `pick` を `squash` (または `s`) に変更:
```
pick abc1234 wip: ログイン機能実装中
squash def5678 wip: 認証部分完了
squash ghi9012 wip: テスト追加

# ↓ 最終的なコミットメッセージを書く
feat: ログイン機能の実装（REQ-XXX）
```

## 注意事項

- WIPコミットは**プッシュ前に整理する**
- `wip:` プレフィックスがあるコミットはPRに含めない
- 長時間作業する場合は**30分〜1時間ごとにWIPコミット**を推奨

## なぜWIPコミットが重要か

1. **作業の損失防止**: クラッシュ・誤操作からの復旧
2. **思考の記録**: 何をやっていたか後から追える
3. **柔軟な整理**: 後からきれいにまとめられる
4. **AI駆動開発との相性**: Claude Codeに続きを頼むときの起点になる
