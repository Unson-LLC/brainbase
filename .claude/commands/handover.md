# セッション引き継ぎ（/handover）

セッション終了前に、次の担当者が5分で状況を把握できる `HANDOVER.md` を生成する。

## 出力先

- 必須: プロジェクトルートの `HANDOVER.md`
- 推奨: 履歴保存として `.claude/handover/HANDOVER_YYYYMMDD_HHMM.md` も作成

## 実行手順

1. 現在の作業状態を確認

```bash
git branch --show-current
git status --short
git diff --stat
git log --oneline -10
```

2. このセッションのやり取り・編集内容・実行結果を要約して、`HANDOVER.md` を作成

3. `HANDOVER.md` に以下のテンプレートを必ず含める

```markdown
# HANDOVER

- Generated At: YYYY-MM-DD HH:mm (TZ)
- Branch: <branch>
- Repo: <repo-path>
- Session Goal: <今回のゴール>

## 1. 今回やったこと（完了）
- 事実ベースで列挙（何を、どこまで）

## 2. 未完了・保留
- 未完了タスク
- ブロッカー
- 保留理由

## 3. 重要な判断と理由
- 判断内容
- 採用理由（トレードオフ）
- 却下した選択肢（あれば）

## 4. 問題・ハマりどころと対処
- 症状
- 原因
- 対処
- 再発防止メモ

## 5. 変更ファイルマップ
- `path/to/file`: 何を変えたか / なぜ重要か / 次にどこを見るか

## 6. 検証結果
- 実行したテスト・コマンド
- 結果（成功/失敗）
- 未実施項目

## 7. 次セッションの最初のアクション（3つ）
1. <最初にやること>
2. <次にやること>
3. <最後にやること>

## 8. 参照情報
- 関連Issue/PR/ドキュメント/ログのパス
```

## 品質ルール

- 推測で埋めない。わからない点は「不明」と書く
- 日付は相対表現ではなく絶対日付（`YYYY-MM-DD`）で書く
- 「次に何をすれば再開できるか」を最優先で明記する
- 未実施のテストや残リスクを隠さない
- 秘密情報（APIキー/トークン/パスワード等）は **絶対に書かない**（必要なら `<REDACTED>` に置換）
