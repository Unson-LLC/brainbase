# Claude Code Hooks データ管理ディレクトリ

## 概要

このディレクトリは、Claude Code Hooks が使用するデータファイル（JSON、状態管理、キャッシュ等）を整理して管理するための専用領域です。

## ディレクトリ構造

```
.claude/hooks/data/
├── auto-context-loader/      # 自動コンテキスト読み込みフック用
│   └── session-state.json   # セッション状態管理
├── quality-checker/          # 品質チェックフック用（将来拡張）
│   └── cache.json           # チェック結果キャッシュ
└── README.md                 # このファイル
```

## 設計原則

### 1. フック固有データの分離

- 各フックが使用するデータは専用ディレクトリで管理
- フック名と同じディレクトリ名を使用

### 2. データファイル命名規則

- **状態管理**: `*-state.json`
- **キャッシュ**: `*-cache.json`
- **設定**: `*-config.json`
- **ログ**: `*-log.json`

### 3. アクセスパターン

```typescript
// フック内でのデータファイルアクセス例
const stateFile = path.join(
  process.cwd(),
  ".claude",
  "hooks",
  "data",
  "フック名",
  "session-state.json",
);
```

## 利点

1. **責任範囲の明確化**: 各フックのデータが明確に分離
2. **メンテナンス性向上**: データの所在が明確
3. **拡張性**: 新しいフック追加時も同じパターンで管理
4. **クリーンアップ容易**: フック削除時にデータも一括削除可能

## 注意事項

- このディレクトリ内のファイルはGit管理対象外（.gitignoreに追加済み）
- 機密情報を含む可能性があるため、共有時は注意
- データファイルは自動生成されるため、手動編集は非推奨
