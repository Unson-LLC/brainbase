# X出荷ライン憲章（SNS Smart Factory）

最終更新: 2026-01-23

## 目的
- 生成量ではなく出荷量（published）を増やす
- 品質を落とさず、毎日安定して流す

## 定義
- **出荷物**: X投稿1本（1ファイル=1投稿）
- **SSOT**: _codex（NocoDBは運用DB）
- **編集パイプライン**: draft → review → ready → posted → archived

## 絶対ルール
- 末尾の「。」禁止（引用/コード内は例外）
- 箇条書き・太字・UIメタ文言禁止
- Pillarに沿った主張を最低1行入れる
- 投稿は「俺/自分の話」にする（他人の職種/現場の断定は書かない）
- CTAは**自然に入る時だけ**（無理に締めない）
- AIに聞く前に一行で問いを書く（書けない日は回さない）
- 全投稿に同一の型を当てない（レーン別ルールで運用）

## 出荷時間帯
- 有効時間: 07:00〜02:00
- 主投稿は3時間以上あける

## 日次KPI（現行）
- 主投稿: 5本/日（**X短文3本 + X記事2本**）
  - **1日5投稿以内ルール準拠**
  - 投稿間隔: 3時間以上
  - 有効時間: 07:00〜02:00（19時間 = 最大6枠、実際は5投稿）
- note: 2本/日（X記事と同一本文）
- 返信/引用: 10〜15本/日

## レーン設計（SNSの揺れを担保）

共通骨格（全投稿必須）
- 俺/自分視点
- 具体1つ（数字 or 固有名詞のどちらか）

レーンA: ドラマ（プロセス/葛藤）
- 必須: 悩み → 判断 → 結果 → 未完了（次の一手）
- 連続性を作る役割（追う理由）

レーンB: 断定（スタンス/価値観）
- 必須: 断定 + 根拠1つ
- 未完了は不要
- 軸足を作る役割

レーンC: 実験ログ（数字/進捗）
- 必須: 数字 + 固有名詞
- 感情は薄くてOK
- 今どこにいるかを伝える役割

レーンD: 共感/日常
- 必須: 温度感 + 小ネタ
- ルール最小
- 人間味を補給する役割

## レーン配分（X短文3本ベース）

**原則**: 1日5投稿以内（X短文3本 + X記事2本）、3時間以上間隔

**X短文3本のレーン配分案**:
| レーン | 配分 | 役割 | 理由 |
|--------|------|------|------|
| **A ドラマ** | 1本 | 返信トリガー | 連続性・追う理由を作る |
| **B 断定** | 1本 | プロフィール遷移 | 軸足・価値観を示す |
| **C 実験ログ** | なし | （X記事で代替） | 数字は長文で深掘り |
| **D 共感/日常** | 1本 | 返信トリガー | 人間味・温度感補給 |

**変動制の運用**:
- 返信重視日: A1 + D2（返信トリガー優先、プロフ遷移なし）
- プロフ遷移日: A1 + B1 + C1（バランス型、実験ログ追加）
- 日によってレーン配分を柔軟に変更可能

**バッチ生産数の根拠**:
- 15本バッチ生成 → QC検証（80点以上） → 5本選定（選定率33%）
- 余裕を持たせて選定品質を担保

## 運用指標
- レビュー滞留が20件を超えたら生成停止
