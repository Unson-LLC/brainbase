# spikes ガイド

ここは **調査・比較・試作・未確定の検証** を置く場所だし。  
まだ正本の設計に昇格していないものは、`architecture/` ではなく基本こっちに置く。

## 置くもの

- 外部プロダクトや OSS の調査
- アダプター接続の検証メモ
- 比較表
- 実装前の試作
- 「この方向で行けるか」の確認メモ

## 置かないもの

- 最終決定
- 正本の設計
- 実装に必要な確定仕様

それらは次に昇格させる。

- 最終決定 → `docs/decisions/`
- 設計の正本 → `docs/architecture/`
- 実装詳細 → `docs/specs/`

## 命名ルール

- ファイル名は `kebab-case`
- 調査・試作であることが分かる接尾辞を優先する
  - `-spike`
  - `-research`
  - `-comparison`
  - `-prototype`
