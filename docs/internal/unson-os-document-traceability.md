# アーキテクチャ: UnsonOS 文書トレーサビリティ

## 文書メタ情報

- 状態: 有効
- 種別: アーキテクチャ
- 主題: UnsonOS 文書系譜管理
- 親文書: [UnsonOS ランタイム UI フレーム](./unson-os-runtime-ui-frame.md)
- 派生元: なし
- 関連文書: [UnsonOS 文書索引](./unson-os-index.md), [UnsonOS 用語ルール](./unson-os-terminology.md)
- 置換: なし

## 目的

`UnsonOS` の文書群を、フォルダ整理だけでなく **系譜が追える状態** で管理するための最小ルールを定義する。

この文書では次を固定する。

- どの文書が親か
- 何から派生したか
- どれが現行の正本か
- 何が関連文書か
- 何が置換済みか

## 基本方針

### 1. フォルダは種類、メタ情報は系譜を表す

- `frames/`, `stories/`, `architecture/`, `specs/`, `spikes/`, `decisions/` は文書種別を表す
- 親子関係や履歴は、各文書の先頭メタ情報で表す

### 2. 同じ主題を複数文書で扱ってよい

ただし、その場合は必ず次を明記する。

- 親文書
- 派生元
- 関連文書
- 置換

### 3. 索引は入口、各文書のメタ情報は正本

- `docs/internal/unson-os-index.md` は読み順と入口を示す
- 文書の系譜は、各文書の `文書メタ情報` を正本とする

## 標準メタ情報

`UnsonOS` 系文書は、タイトル直下に次の 7 項目を置く。

- `状態`
- `種別`
- `主題`
- `親文書`
- `派生元`
- `関連文書`
- `置換`

## 項目の意味

### 状態

- `有効`: 現行の正本
- `草案`: 叩き台
- `検証中`: スパイクや仮説検証中
- `置換済み`: 新文書に置き換え済み
- `保留`: 現行判断には使わない

### 種別

- `フレーム`
- `ストーリー`
- `アーキテクチャ`
- `仕様`
- `スパイク`
- `決定`

### 主題

文書が扱う主題を短く書く。

### 親文書

この文書の判断基準になっている上位文書。

### 派生元

この文書を起こすきっかけになった文書や議論。

### 関連文書

同じ主題の横展開や、前後で必ず読む文書。

### 置換

次のどちらかを書く。

- `なし`
- 置き換えた旧文書
- この文書を置き換える新文書

## 運用ルール

### 1. 新規文書を作るとき

- 先に `主題` を決める
- 次に `種別` を決める
- 最後に `親文書` と `派生元` を決める

### 2. 文書を分岐させるとき

派生した側は必ず `派生元` に元文書を書く。

### 3. 文書を置き換えるとき

- 旧文書の `状態` を `置換済み` にする
- `置換` に新文書を書く
- 新文書の `派生元` に旧文書を書く

### 4. スパイクは正本にしない

- `spikes/` は検証中として扱う
- 採用した内容は `architecture/` か `decisions/` に昇格する

## 最低限の見方

### 本流を追うとき

1. フレーム
2. ストーリー
3. アーキテクチャ
4. 仕様

### 議論の系譜を追うとき

1. `派生元`
2. `親文書`
3. `関連文書`
4. `置換`
