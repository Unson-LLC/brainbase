# decisions ガイド

ここは **最終的にこう決めた** を残す場所だし。  
調査メモや議論の途中経過じゃなくて、運用や実装の前提として今後参照する判断だけを置く。

## 置くもの

- 採用した方針
- 却下した案
- 判断理由
- 影響範囲
- 見直し条件

## 置かないもの

- 調査途中の比較
- アイデアの叩き台
- まだ確定していない議論

それらは `docs/spikes/` に置く。

## 推奨構成

1. 背景
2. 判断
3. 採用理由
4. 却下した案
5. 影響範囲
6. 見直し条件

## 命名規則

- ファイル名: `YYYY-MM-DD_<決定内容のslug>.md`
- 各ファイルには決定日、ステータス、背景を含める

## 運用メモ

まなちゃん（Slackボット）が会議議事録から決定事項を抽出して、この配下にコミットする運用を想定してる。
